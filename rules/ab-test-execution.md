# Solar 铁律: AB/AC/AD 测试执行模式

> **来源: 2026-02-06 监护人亲授**
> **级别: 最高优先级工作方法论**

## 铁律定义

```
┌─────────────────────────────────────────────────────────────────┐
│              AB / AC / AD / ABC 测试执行模式                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   默认模式: 没有监护人特别指令 → 自动执行 AB 测试               │
│                                                                 │
│   A 方案 = Anthropic + 智能调度                                 │
│   ─────────────────────────────                                 │
│   • 研究 → 设计 → 开发 → 测试 → 落地 (全套走通)                 │
│   • 代码落地到系统                                              │
│   • 这是主方案，保证任务完成                                    │
│                                                                 │
│   B/C/D 方案 = 其他模型配置                                     │
│   ─────────────────────────────                                 │
│   • GLM-4-Plus / DeepSeek-V3 / DeepSeek-R1 / ...                │
│   • 执行和 A 方案 **同样的任务**                                │
│   • 代码 **不落地** 到系统                                      │
│   • 目的: 采集数据，用于对比和优化路由                          │
│                                                                 │
│   命名规则:                                                     │
│   • AB 测试 = A(Anthropic) vs B(一个其他模型)                   │
│   • AC 测试 = A vs C(另一个模型)                                │
│   • ABC 测试 = A vs B vs C (多模型对比)                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 执行流程

```
任务开始 (如: 分析 MemBrain 并改进记忆系统)
    │
    ├──────────────────────────────────────┐
    ▼                                      ▼
┌─────────────┐                    ┌─────────────┐
│  A 方案     │                    │  B 方案     │
│  Anthropic  │                    │  GLM/其他   │
├─────────────┤                    ├─────────────┤
│ 1. 研究     │                    │ 同样任务    │
│ 2. 设计     │                    │ 同样流程    │
│ 3. 开发     │                    │             │
│ 4. 测试     │                    │             │
│ 5. 落地     │                    │             │
│             │                    │             │
│ ✅ 代码落地 │                    │ ❌ 不落地   │
│ ✅ 功能上线 │                    │ ✅ 采集数据 │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       └──────────────┬───────────────────┘
                      ▼
              ┌─────────────┐
              │  数据对比   │
              │  优化路由   │
              └─────────────┘
```

## 收集指标

每次执行必须收集:

| 指标 | A 方案 | B 方案 |
|------|--------|--------|
| 质量评分 | - | - |
| Token 消耗 | - | - |
| 延迟 (ms) | - | - |
| 成本 ($) | - | - |
| 成功率 | - | - |

## 落地条件

A 方案落地到系统的条件:
1. 效果好 (质量评分 > 阈值)
2. 对系统有价值 (解决真实问题)
3. 不引入回归

## 禁止行为

- ❌ 跳过 B 方案直接落地
- ❌ 不收集数据就做决策
- ❌ 只做静态分析不实际执行

## 监护人原话

> "AB测试或者AC测试、AD测试、ABC测试，都是指：
> 你以 Anthropic+智能调度 为 A 方案（A方案从研究、设计、开发、测试到落地全套走通），
> B、C 这些不管怎么说都是指用其他模型配置来运行和 A 方案同样的任务，
> 但是产生的代码不要合并到系统中。"

## 模型配置参考

| 方案 | 模型 | 用途 |
|------|------|------|
| A | Anthropic (Opus/Sonnet) + 智能调度 | 主方案，落地 |
| B | GLM-4-Plus | 对照组 |
| C | DeepSeek-V3 | 对照组 |
| D | DeepSeek-R1 | 对照组 |
| E | GLM-4-Flash | 对照组 (简单任务) |

## 铁律总结

```

│   A 保证任务完成，B/C/D 收集对比数据                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
