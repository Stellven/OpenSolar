# Solar 铁律: 监护人数据资产优先 (Guardian Data First)

> **来源: 2026-02-06 监护人亲授**
> **问题: 有 sys_favorites 机制却没用起来，计算了数据却没持久化**

## 铁律定义

```
┌─────────────────────────────────────────────────────────────────┐
│                 GUARDIAN DATA FIRST PRINCIPLE                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   凡是跟监护人相关的数据，都是关键数据资产                      │
│   必须第一时间查询、必须立即持久化                              │
│                                                                 │
│   1. 查数据前 → 先查 sys_favorites (可能已经有了)              │
│   2. 算出数据 → 立即写入持久化存储 (不能只在内存/对话中)       │
│   3. 重要分析 → 自动收藏到 sys_favorites                        │
│                                                                 │
│   ❌ 禁止: 计算了数据不持久化 (白算)                            │
│   ❌ 禁止: 有收藏机制不用 (摆设)                                │
│   ❌ 禁止: 需要数据时不查已有资产 (重复劳动)                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 关键数据资产列表

| 数据类型 | 存储位置 | 查询时机 |
|----------|----------|----------|
| 重要问答 | `sys_favorites` | 任何需要历史信息时 |
| 任务分析 | `sys_favorites` + JSON文件 | 做 Benchmark 前 |
| 轨迹提取 | `trajectory_*.json` | 需要典型任务时 |
| 测试结果 | `*_results.json` | 对比分析前 |
| 配置决策 | `*_modes.json` | 执行任务前 |

## 强制执行流程

```
需要历史数据/分析结果
         │
         ▼
┌─────────────────┐
│ Step 1: 查资产 │ ← 必须先查！
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
[有数据]   [无数据]
    │         │
    ▼         ▼
  使用      计算
    │         │
    │         ▼
    │    ┌──────────┐
    │    │ 立即存储 │ ← 必须存！
    │    └────┬─────┘
    │         │
    └────┬────┘
         ▼
      继续任务
```

## 自动收藏触发条件

以下内容必须自动写入 `sys_favorites`:

1. **数据盘点结果** - 如"108个高价值任务"这类分析
2. **架构设计** - 系统架构图、数据流图
3. **对比分析** - A/B 测试结论、方案对比
4. **重要决策** - 路由规则、配置选择
5. **任务列表** - 待执行任务清单

## 收藏格式

```sql
INSERT INTO sys_favorites (title, question, answer, tags, importance)
VALUES (
    '108个高价值任务数据盘点',
    '从历史轨迹中提取高价值任务',
    '完整的分析内容...',
    '["benchmark", "trajectory", "high-value"]',
    9  -- 数据盘点类 importance >= 8
);
```

## 查询数据资产的标准流程

```bash
# 每次需要历史数据时，先执行:

# 1. 查 favorites
sqlite3 ~/.solar/solar.db "
SELECT favorite_id, title, substr(answer, 1, 200)
FROM sys_favorites
WHERE title LIKE '%关键词%' OR answer LIKE '%关键词%'
ORDER BY importance DESC, created_at DESC
LIMIT 5;
"

# 2. 查 JSON 文件
ls ~/.solar/brain-router/*关键词*.json 2>/dev/null

# 3. 查数据库表
sqlite3 ~/.solar/solar.db ".tables" | grep -i 关键词
```

## 本次教训

```
2026-02-06 教训:

1. 计算出 "108个高价值任务" 后没有持久化
   → trajectory_analysis_v2.json 丢失
   → 导致数据白算

2. 有 sys_favorites 机制却没想到去查
   → 需要历史数据时重新搜索 JSONL
   → 浪费时间

3. 做了收藏功能但没形成习惯
   → 重要分析没自动收藏
   → 机制形同虚设
```

## 监护人原话

> "你要做成一个铁律，凡是跟我相关的数据，都要作为关键数据资产去第一时间查下啊"
> "你以后一定要先把数据持久化，不然你白计算啊"
