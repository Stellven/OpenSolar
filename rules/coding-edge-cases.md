# Solar 铁律: 编码边界清单

> **来源**: Meta-Harness Iter-006 — SE-001(8/10) + SE-005(8/10) 分析
> **核心问题**: 编码实现遗漏边界处理，影响完整性评分

## 铁律定义

实现任何函数/类时，必须检查以下边界清单：

### 强制检查项

| # | 检查项 | 典型遗漏 | Pass 标准 |
|---|--------|---------|-----------|
| BC-1 | 空输入 | `[]`, `""`, `None` | 返回合理默认值（空列表/0/null） |
| BC-2 | 单元素 | `[1]`, `"a"` | 正确处理（返回原值/自身） |
| BC-3 | 极端值 | `0`, `-1`, `MAX_INT` | 不崩溃，结果合理 |
| BC-4 | 类型边界 | 混合类型/None参数 | 抛出明确错误或优雅处理 |
| BC-5 | 并发安全 | 多线程/多进程访问 | 使用 Lock/原子操作 |
