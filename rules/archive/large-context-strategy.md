## 策略转变

| 旧策略 (错误) | 新策略 (正确) |
|--------------|--------------|
| 担心上下文长度 | 200K 足够用 |
| 压缩、精简 | 直接放、不压缩 |
| 靠"记得查" | 靠"已经看到" |
| Hook 提醒 | 上下文预加载 |
| 规则存在文件里 | 规则加载到上下文 |

## 实现: SessionStart 预加载

**context-preload.sh** 在每次会话开始时加载：

1. **搜索引擎状态**
   - Tantivy daemon 是否运行
   - 索引文档数量

2. **数据资产清单**
   - 按类型分组统计
   - 记录数量

3. **已有能力清单**
   - Skills: 85 个
   - Shortcuts: 61 个
   - Tools: 32 个
   - Agents: 10 个

4. **搜索入口**
   - Tantivy 命令
   - FTS5 命令
   - SQL 查询

5. **最近能力 (7天内)**
   - 防止重复创建刚做过的

## 为什么有效？

```
旧方案: Hook 提醒
─────────────────────────
用户说"建索引"
  → Hook 输出"请先查是否有"
  → 我可能忽略提醒
  → 还是直接开始建

新方案: 上下文预加载
─────────────────────────
会话开始
  → 数据资产清单已在上下文
  → 我看到"Tantivy: 15513 文档"
  → 用户说"建索引"
  → 我自然会想到"已经有 Tantivy 了"
```

## 200K 上下文容量规划

| 内容 | 估算 Token | 优先级 |
|------|-----------|--------|
| CLAUDE.md | ~3K | 必须 |
| 活跃规则 | ~5K | 必须 |
| 数据资产清单 | ~1K | 必须 |
| 已有能力列表 | ~2K | 必须 |
| 最近记忆 | ~5K | 推荐 |
| 当前任务上下文 | ~10K | 按需 |
| 对话历史 | ~50K | 自动 |
| **总计** | ~76K | **< 200K** |

**结论**: 还有 120K+ 余量，完全不需要压缩！
