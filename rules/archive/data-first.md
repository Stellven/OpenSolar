# Solar 铁律: 数据是第一生产力

> **来源: 2026-02-04 监护人指示**
> **教训: 忽视了 99% 的可用数据**

## 铁律定义

```
┌─────────────────────────────────────────────────────────────────┐
│                    DATA FIRST PRINCIPLE                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   任何需要数据的任务，必须:                                     │
│                                                                 │
│   1. 先盘点: 有什么数据？                                       │
│   2. 再设计: 如何利用？                                         │
│   3. 最大化: 用尽可用数据                                       │
│                                                                 │
│   ❌ 禁止: 设计先行，等待理想数据                               │
│   ❌ 禁止: 忽视现有数据，用默认值凑数                           │
│   ❌ 禁止: 数据利用率 < 50%                                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 教训来源

2026-02-04 计算 B 人格时:
- 数据库有 3400+ 条数据
- 我只用了 31 条 (< 1%)
- 原因: 按预设设计查 evo_tool_calls (空表)
- 忽视: hn_topics (1719), smi_files (1527), bl_tasks (54)

## 正确流程

```
任务: 需要数据支撑的分析/计算/决策
         │
         ▼
Step 1: 全局盘点
─────────────────────────────────────
sqlite3 db "SELECT name, (SELECT COUNT(*) FROM ...)
            FROM sqlite_master WHERE type='table'"
         │
         ▼
Step 2: 识别可用数据
─────────────────────────────────────
• 哪些表有数据？
• 数据量各多少？
• 数据结构是什么？
         │
         ▼
Step 3: 设计利用方案
─────────────────────────────────────
• 每个数据源能提供什么信息？
• 如何组合多个数据源？
• 如何处理数据缺失？
         │
         ▼
Step 4: 最大化利用
─────────────────────────────────────
• 利用率目标: > 80%
• 如果某数据源没用到，问自己为什么
```

## 快速盘点命令

```bash
# 一键盘点所有表数据量
for t in $(sqlite3 ~/.solar/solar.db ".tables" | tr ' ' '\n'); do
  c=$(sqlite3 ~/.solar/solar.db "SELECT COUNT(*) FROM $t" 2>/dev/null)
  [ "$c" -gt 0 ] && printf "%s\t%s\n" "$t" "$c"
done | sort -t$'\t' -k2 -rn | head -20
```

## 强制执行机制 (端到端保证)

### 1. 数据账本 (sys_data_assets)

```sql
-- 查看数据资产账本
SELECT asset_id, record_count, description
FROM sys_data_assets
ORDER BY record_count DESC;
```

### 2. 任务前必查

**每次开始需要数据的任务前，必须执行:**

```bash
~/Solar/core/ontology/data-inventory.sh
```

### 3. Session Start Hook 检查

SessionStart Hook 会显示数据资产摘要，提醒可用数据。

### 4. 人格学习器强制盘点

`personality-learner.sh` 开头必须调用 `data-inventory.sh`，
确保在计算人格前知道有哪些数据可用。

### 5. 自我检测触发词

当我说以下词时，必须先盘点数据:
- "计算人格"
- "分析数据"
- "生成报告"
- "统计"
- "根据数据"

### 6. 违反检测

如果输出包含以下模式，说明违反了铁律:
- "没有数据" → 立即检查数据账本
- "用默认值" → 是否真的没有数据？
- "只有 X 条" → 是否遗漏了其他数据源？

## 数据账本快速查看

```bash
# 一键查看所有数据资产
sqlite3 ~/.solar/solar.db "
SELECT
    asset_type as '类型',
    COUNT(*) as '资产数',
    SUM(record_count) as '记录数',
    SUM(size_bytes)/1024/1024 || ' MB' as '大小'
FROM sys_data_assets
GROUP BY asset_type;
"
```
