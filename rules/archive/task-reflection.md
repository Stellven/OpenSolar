# Solar 铁律: 任务反思机制

> **来源: 2026-02-04 监护人指示**
> **强制执行: 每次端到端任务完成后必须反思**

## 铁律定义

```
┌─────────────────────────────────────────────────────────────────┐
│                    TASK REFLECTION PROTOCOL                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   每次完成一个端到端任务后，必须执行反思：                      │
│                                                                 │
│   1. 提炼经验 - 哪些做得好？哪些可以改进？                      │
│   2. 发现缺失 - 缺少什么工具/技能/Agent？                       │
│   3. 提出建议 - 哪些应该加入任务列表？                          │
│   4. 让监护人决定 - 是否加入 Backlog                           │
│                                                                 │
│   ❌ 禁止: 完成任务后直接结束                                   │
│   ✅ 必须: 输出结构化的反思报告                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 什么是"端到端任务"

| 是端到端任务 | 不是端到端任务 |
|-------------|----------------|
| 实现完整的功能/系统 | 读取一个文件 |
| 修复一个 Bug (含测试验证) | 回答一个问题 |
| 完成一个 Feature 的所有任务 | 执行单个工具调用 |
| 从设计到部署的完整流程 | 中间步骤 |

**判断标准:** 是否是一个完整的、可交付的成果

## 执行时机

**触发条件:**
1. 用户明确表示任务完成 (如说"好"/"可以")
2. 达成一个里程碑 (如完成整个 Feature)
3. 部署上线一个系统
4. 修复并验证一个重要 Bug

**执行流程:**
```
任务完成
    │
    ▼
输出反思报告 (使用模板)
    │
    ▼
等待监护人反馈
    │
    ├─ 同意 → 加入 Backlog
    └─ 调整 → 修改建议后加入
```

## 反思的深度

| 任务规模 | 反思深度 | 建议数量 |
|---------|---------|---------|
| 小型 (<1小时) | 简要反思 | 1-2 条建议 |
| 中型 (1-4小时) | 标准反思 | 3-5 条建议 |
| 大型 (>4小时) | 深度反思 | 5-10 条建议 |

## 反思的质量标准

**好的反思:**
- ✅ 具体可行 - "缺少文件监听工具"
- ✅ 有理有据 - "因为手动更新索引太慢"
- ✅ 优先级明确 - "这个是高优先级，因为..."
- ✅ 可量化 - "节省 80% 的搜索时间"

**不好的反思:**
- ❌ 太笼统 - "需要优化性能"
- ❌ 无依据 - "感觉应该做..."
- ❌ 无优先级 - 全部都说重要
- ❌ 无法执行 - "让系统更智能"

## 反思数据存储

**存储位置:** `~/.solar/reflections/`

**文件命名:** `YYYY-MM-DD_task-name.md`

**结构化数据:** 同时写入数据库 `ont_learning_events` 表

```sql
INSERT INTO ont_learning_events (
  event_type,
  content,
  confidence,
  evidence
) VALUES (
  'task_reflection',
  '反思内容',
  0.9,
  '任务完成证据'
);
```

## 与本体系统的关联

反思结果会自动更新：
1. **Procedural Memory** - 新的工作流程
2. **Semantic Memory** - 新的知识点
3. **偏好更新** - 发现的偏好模式

## 检查清单

每次任务完成后，自问：

- [ ] 这是一个端到端的完整任务吗？
- [ ] 我用了什么新方法/工具？
- [ ] 有什么可以做得更好？
- [ ] 缺少什么工具/技能？
- [ ] 哪些经验可以复用？
- [ ] 有什么应该固化成规则？
- [ ] 监护人可能关心什么指标？
