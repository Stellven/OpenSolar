# Solar 铁律: 多专家会审 (Multi-Expert Analysis)

> **来源: 2026-02-07 监护人亲授**
> **核心: 分析阶段要集思广益，专家组团会审**

## 铁律定义

```
┌─────────────────────────────────────────────────────────────────┐
│              MULTI-EXPERT ANALYSIS PROTOCOL                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   分析阶段 ≠ 单专家意见                                         │
│   分析阶段 = 多专家组团会审                                     │
│                                                                 │
│   ❌ 禁止: 分析任务只调用一个老专家                             │
│   ✅ 必须: 至少 2-3 个老专家并行分析，综合意见                  │
│                                                                 │
│   监护人原话:                                                   │
│   "一般分析阶段，你要集思广益啊，要专家组团会审"                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 什么是"分析阶段"

| 是分析任务 | 不是分析任务 |
|-----------|-------------|
| 技术方案评估 | 简单代码生成 |
| 架构设计 | 格式转换 |
| 文章/论文解读 | 数据查询 |
| 可行性研究 | 文件读写 |
| 新技术调研 | 测试执行 |
| 方案对比 | 简单问答 |

**判断标准**: 需要深度思考、需要多角度视角、结论影响决策

## 专家组合建议

### 深度分析类 (技术调研、论文解读)

```
思考驼 (deepseek-r1)   - 深度推理、本质洞察
千里马 (gemini-3-pro)  - 创新方案、多维权衡
鬼才码农 (deepseek-v3) - 实现思路、代码结构
```

### 架构设计类

```
技术宅 (gemini-2.5-pro) - 严谨审查、一致性
千里马 (gemini-3-pro)   - 创新探索、方案设计
思考驼 (deepseek-r1)    - 深层分析、风险评估
```

### 快速评估类 (简单方案对比)

```
老实人 (glm-4-plus)  - 日常分析
闪电侠 (gemini-flash) - 快速总结
```

## 执行流程

```
分析任务到达
       │
       ▼
┌─────────────────┐
│ 判断是否需会审  │
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
[简单分析]  [深度分析]
    │         │
    ▼         ▼
 2个专家    3个专家
 并行调用   并行调用
    │         │
    └────┬────┘
         ▼
   综合专家意见
         │
         ▼
   输出会审结论
```

## 调用方式

```typescript
// ❌ 错误: 只调一个
await mcp__brain_router__complete({
  model: "deepseek-r1",
  prompt: "分析这个技术..."
});

// ✅ 正确: 并行调用多个
await Promise.all([
  mcp__brain_router__complete({
    model: "deepseek-r1",
    system: "你是思考驼，擅长深度推理...",
    prompt: "分析这个技术的本质和风险..."
  }),
  mcp__brain_router__complete({
    model: "gemini-3-pro-preview",
    system: "你是千里马，擅长创新探索...",
    prompt: "分析这个技术的应用方案..."
  }),
  mcp__brain_router__complete({
    model: "deepseek-v3",
    system: "你是鬼才码农，擅长实现思路...",
    prompt: "分析这个技术的代码实现..."
  })
]);
```

## 自检清单

收到分析任务时，问自己:

- [ ] 这是需要深度思考的分析任务吗？
- [ ] 我是不是只打算调用一个专家？
- [ ] 应该用哪几个专家组团会审？
- [ ] 他们各自的分析角度是什么？

## 违反案例 (本次教训)

```
任务: 分析 MLX 优化
错误: 只调用了思考驼一个专家
正确: 应该并行调用 思考驼 + 千里马 + 鬼才码农

用户反馈:
"你这个只用一个，我觉得不好"
```
