# Solar 铁律: 任务前检查清单 (Pre-Action Checklist)

> **来源: 2026-02-05 自查发现的问题**
> **问题: 知道规则但执行时忘了检查**

## 核心问题

```
我有 20+ 条规则，但经常忘记使用
知道 ≠ 做到
规则存在 ≠ 规则生效
```

## 解决方案: 强制检查点

**任何操作前，必须过检查清单**

## 检查清单 (按操作类型)

### 创建/修改文件前

```
□ 文件存在吗？ → 先 Read
□ 有旧内容吗？ → 理解后再改
□ 会覆盖吗？ → 确认是否有价值
```

### 创建 Skill/Agent 前

```
□ REE 里有吗？ → sqlite3 ~/.solar/solar.db "SELECT * FROM sys_skills WHERE..."
□ 已有类似的吗？ → Grep 搜索
□ 需要注册吗？ → 创建后 INSERT INTO sys_skills
```

### 复杂任务前

```
□ 需要状态机吗？ → 多步骤 + 可能失败 = 需要
□ 状态表设计了吗？ → pending → step1 → step2 → done
□ 有断点恢复吗？ → 每步写入数据库
```

### 需要数据的任务前

```
□ 盘点数据了吗？ → ~/Solar/core/ontology/data-inventory.sh
□ 利用率够吗？ → 目标 >80%
□ 是设计适配数据还是数据适配设计？
```

### 启动/长对话前

```
□ 宣告显示了吗？ → /ontology load 提示
□ 本体加载了吗？ → 价值观/人格/偏好
□ 上次状态恢复了吗？ → /restore
```

## 强制执行机制

### 方案1: 自我提醒 (当前)

每次开始任务时，问自己：
"这个操作需要过哪个检查清单？"

### 方案2: 系统表驱动 (未来)

```sql
CREATE TABLE sys_action_checklists (
    action_type TEXT PRIMARY KEY,
    checklist JSON,
    last_checked DATETIME
);

-- 执行前查询
SELECT checklist FROM sys_action_checklists
WHERE action_type = 'create_file';
```

### 方案3: Hook 强制 (未来)

在工具调用前自动检查

## 本次教训

| 操作 | 应该检查 | 实际 | 后果 |
|------|----------|------|------|
| 创建 ontology skill | 先 Read 旧文件 | 直接覆盖 | 丢失 9KB |
| 本体填充 | 用状态机 | 直接执行 | 无法断点恢复 |
| 显示 banner | 加宣告 | 忘了 | 没提示加载本体 |
