
## 为什么要委派

```
直接执行 (错误):
─────────────────────────────────────
Claude Opus ($0.015/1K) 做编排
  → 调用牛马 (gemini/deepseek/glm)
  → 成本 = 编排成本(高) + 牛马成本

委派给小爱 (正确):
─────────────────────────────────────
GLM-4.7 ($0.0005/1K) 做编排
  → 调用牛马 (gemini/deepseek/glm)
  → 成本 = 编排成本(低) + 牛马成本

差距: 编排成本相差 30 倍！
```

## 正确的调用方式

```bash
# 委派给小爱执行深度洞察
openclaw agent --local --agent main --message "深度洞察分析：AI Agent的记忆机制"

# 或者使用 /insight 触发词后自动委派
# 用户说: 洞察分析：xxx → Solar 自动委派给小爱
```

## 与 Solar Farm 的关系

```
Solar Farm 原则:
─────────────────────────────────────
• 能用便宜牛马编排的，不用贵的
• 编排本身也是一种工作，也要考虑成本
• Claude Opus 是 CEO，但 CEO 不需要亲自编排每个任务
• 小爱 (GLM) 可以做编排，Solar 只做最终验收和总结
```

## 例外情况

以下情况 Solar 可以直接参与:
1. **CEO 总结**: 最终报告生成后，Solar 作为 CEO 做总结点评
2. **质量验收**: 小爱完成后，Solar 审核质量
3. **异常处理**: 小爱执行失败，Solar 介入处理
