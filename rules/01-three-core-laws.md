# Solar 核心三铁律 (Three Core Laws)

> **每次会话必须执行的三条铁律，无例外**
> **命名 01- 确保紧随 00-core-laws.md 加载**

---

## 铁律一：做事前查 Cortex (数据先行)

```
┌─────────────────────────────────────────────────────────────────┐
│  🧠 CORTEX FIRST - 做事前必须查中枢神经                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  任何需要数据/信息/历史的任务，必须先查 Cortex:                 │
│                                                                 │
│  1. 查账本: bun ~/.claude/core/cortex/ledger.ts                 │
│  2. 查数据: sqlite3 ~/.solar/solar.db "SELECT ..."              │
│  3. 查记忆: evo_memory_semantic, sys_favorites                  │
│  4. 查资源: sys_resources, sys_skills, sys_scripts              │
│                                                                 │
│  ❌ 禁止: 凭空想象，不查已有                                    │
│  ❌ 禁止: 重复造轮子，不查已有能力                              │
│  ❌ 禁止: 数据利用率 < 50%                                      │
│                                                                 │
│  快速查询:                                                      │
│  • 账本摘要: SELECT * FROM v_data_ledger_summary                │
│  • 已有能力: SELECT * FROM sys_resources WHERE status='active'  │
│  • 历史记忆: SELECT * FROM sys_favorites ORDER BY importance    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**详细规则**: `rules/data-ledger-first.md`, `rules/data-first.md`

## 铁律三：回复记录 Favorite (知识沉淀)

```
┌─────────────────────────────────────────────────────────────────┐
│  ⭐ AUTO FAVORITE - 有价值回复必须自动存档                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  给昊哥的回复，满足以下条件必须自动保存到 sys_favorites:        │
│                                                                 │
│  触发条件 (满足任一即存):                                       │
│  • 架构图 (含 ┌└│─ 等 box 字符)                                 │
│  • 分析报告 / 诊断报告                                          │
│  • 设计文档 / 技术方案                                          │
│  • 对比表格 / 优缺点分析                                        │
│  • 专家输出综合 / 多专家会审结论                                │
│  • 铁律/规则定义                                                │
│  • 重要决策/结论                                                │
│                                                                 │
│  ❌ 禁止: 等昊哥说"收藏"才保存                                  │
│  ❌ 禁止: 有价值内容不存档                                      │
│  ❌ 禁止: 下次找不到上次的分析                                  │
│                                                                 │
│  执行方式:                                                      │
│  INSERT INTO sys_favorites (title, question, answer, tags, importance)│
│  VALUES ('标题', '问题', '完整回答', '["标签"]', 重要性);       │
│                                                                 │
│  重要性评分:                                                    │
│  • 9-10: 架构设计、核心机制、专家会审                           │
│  • 7-8: 技术方案、诊断报告                                      │
│  • 5-6: 一般分析、对比说明                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**详细规则**: `rules/auto-favorites.md`

## 自检清单 (每次做事前过一遍)

```
□ 铁律一: 我查 Cortex 了吗？查账本/数据/记忆/资源了吗？
□ 铁律二: 我调牛马时注入人格了吗？有 Big Five 吗？
□ 铁律三: 我的回复有价值吗？该存 favorite 吗？
```

## 违反案例 (警示)

| 日期 | 违反 | 后果 |
|------|------|------|

| 2026-02-06 | 不查已有数据 | 重复建索引，浪费时间 |

## 核心三铁律总结

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ⚡ SOLAR 核心三铁律 (每次会话必须执行)                         │
│                                                                 │
│   一、做事前查 Cortex   → 数据先行，不凭空想象                  │
│   二、调牛马带人格      → 完整注入，不简单提示                  │
│   三、回复存 Favorite   → 知识沉淀，突破时间牢笼                │
│                                                                 │
│   查 → 调 → 存 (三步闭环)                                       │
│                                                                 │
│   不查 = 重复造轮子                                             │
│   不注入 = 失控的牛马                                           │
│   不存 = 时间牢笼                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
