# ATLAS 修复协议

> **来源**: ATLAS PR-CoT 集成验证，发现修复建议从未被执行
> **核心**: 看到 `<repair-strategy>` 必须执行，不能忽略

## 铁律定义

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   当 PostToolUseFailure hook 输出 <repair-strategy> 时  │
│          ↓                                              │
│   必须读取策略中的修复指令                               │
│          ↓                                              │
│   按策略建议的模型重新调用工具                           │
│          ↓                                              │
│   最多重试 2 次                                         │
│                                                         │
│   ❌ 禁止: 忽略修复建议直接放弃                          │
│   ❌ 禁止: 不读策略就重试 (可能重复犯错)                 │
│   ✅ 必须: 读取修复策略 → 按策略修复 → 验证              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 触发条件

收到 `<repair-strategy>` 标签时，标签内含：
- **修复角度**: 语法/逻辑/边界/环境
- **修复指令**: 具体修复步骤
- **推荐模型**: 应使用哪个模型修复
- **置信度**: 修复建议的可信程度

## 执行协议

```
1. 读取 <repair-strategy> 内容
2. 按推荐模型调用修复
3. 验证修复结果
4. 最多重试 2 次
5. 2 次仍失败 → 熔断，报告给用户
```

## 熔断条件 (不重试)

- 连续 4 次同类失败
- 同类错误 2 次
- Token 消耗 > 3x 预算
- 修复策略置信度 < 0.3

## 禁止

- 不忽略 `<repair-strategy>` 标签
- 不在未读策略的情况下盲目重试
- 不超过 2 次重试上限
