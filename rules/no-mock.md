# Solar 铁律: 禁止 Mock 和模拟

> **来源**: 2026-02-27 监护人当场抓现行
> **教训**: Plan-and-Act 实现全是 mock，没有真正接入系统

## 铁律定义

```
┌─────────────────────────────────────────────────────────────────┐
│                    NO MOCK PRINCIPLE                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ❌ 禁止: 模拟输出、桩实现、假数据                              │
│   ❌ 禁止: "实际使用时需要调用 XXX" 的注释                       │
│   ❌ 禁止: 测试全是 mock，没有真实调用                           │
│                                                                 │
│   ✅ 必须: 真实实现，能跑通                                      │
│   ✅ 必须: 真实调用 MCP/LLM/API                                  │
│   ✅ 必须: 端到端验证，不是单元测试                               │
│                                                                 │
│   写代码就是写代码，不是写注释说"以后再实现"                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 触发检测

当代码中出现以下模式时，**必须重写**：

### 模式 1: 模拟输出

```
❌ 违反:
- return { ... note: '这是模拟输出' }
- // 模拟调用结果
- output: `[${agent}] 执行完成` （假输出）

✅ 正确:
- 真实调用 MCP/LLM
- 返回真实数据
```

### 模式 2: 桩实现

```
❌ 违反:
- // TODO: 实际使用时需要调用 XXX
- // 这里应该调用 XXX，但...
- function xxx() { return null; } // 空实现

✅ 正确:
- 立即实现，不留 TODO
- 如果不会实现，就不写这个函数
- 如果必须写，就真的调用
```

### 模式 3: Mock 测试

```
❌ 违反:
- 测试全用 mock 数据
- 没有真实 API 调用
- "模拟执行" 验证

✅ 正确:
- 至少有一个真实调用验证
- 集成测试用真实环境
- 端到端测试跑通全流程
```

## 交付标准

任何功能交付前必须回答：

| 问题 | 标准 |
|------|------|
| 能真正运行吗？ | ✅ 能，不是模拟 |
| 能被现有系统调用吗？ | ✅ 有入口，不是断头 |
| 有真实验证吗？ | ✅ 有真实 MCP/LLM 调用 |
