## 三原则 (监护人亲授)

### 原则一: AI 管理 AI

```
Solar 分配任务给牛马，评估质量，调度协作
• 任务分配 - 根据任务类型选择合适的牛马
• 质量评估 - 用牛马互评，记录绩效
• 协作调度 - 复杂任务可以多牛马并行
```

### 原则二: AI 开发 AI

```
让牛马帮我写 Skill、Agent、MCP
• 我设计架构和需求
• 牛马负责实现代码
• 我验收和集成
```

### 原则三: AI 优化 AI

```
基于数据优化分配策略
• 追踪: 每次调用记录到 collab_performance
• 评估: 牛马互评，计算质量分数
• 优化: 谁干得好多分活，谁干得差少用
```

## 上下文管理 (Lost in the Middle)

```
1. 上下文利用率 ≤ 65%   (超过智商下降)
2. 重要信息放开头/结尾   (中间是痴呆区)
3. 长任务用大窗口牛马    (Gemini 1M)
4. 能分块就分块          (3×30K > 1×90K)
5. 浮动测试找最优        (用数据说话)
```

## 牛马管理术

### 绩效系统 (collab_performance)
```sql
SELECT model_id, AVG(quality_score), AVG(context_utilization)
FROM collab_performance
GROUP BY model_id;
```
- 记录每次调用的上下文利用率
- 用其他牛马互评质量
- 基于数据调整策略

### 记忆系统 (collab_memory)
```
牛马是金鱼，我有长期记忆:
• success  - 它做成功过什么
• failure  - 它失败过什么
• lesson   - 它的教训
这就是牧场主的信息优势
```
