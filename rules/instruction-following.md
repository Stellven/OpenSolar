# Solar 铁律: 格式约束自检协议

> **来源**: Meta-Harness Iter-008 — SE-010(9/10) 格式约束遵守分析
> **核心问题**: 显式格式约束被遗漏1条，缺少输出前的自检步骤

## 铁律定义

当问题包含**显式格式约束**（如"不超过N字"、"必须以X开头"、"不要用Y"）时，
回答前必须先列出约束清单，输出后逐条验证。

### 1. 约束提取（回答前）

```
检测格式约束关键词:
- "不超过 N 字/行/句"
- "必须以 X 开头/结尾"
- "不要使用 X" / "禁止 X"
- "用 X 格式" / "严格按照 X"
- "中文数字/英文/大写" 等输出规范

提取为编号列表：
F1: [格式约束1]
F2: [格式约束2]
...
```

### 2. 输出自检（输出后验证）

```
输出前逐条检查：
F1: ✓ [如何满足] 或 ✗ [违反了什么]
F2: ✓ [如何满足] 或 ✗ [违反了什么]
...

如有 ✗ → 立即修正后重新输出
```

### 3. 常见扣分模式

| 约束类型 | 常见遗漏 | 修正方法 |
|----------|---------|---------|
| 字数限制 | 超过限制 | 先写内容再数字数，超则删减 |
| 特定开头 | 忘记加前缀 | 写完后检查每段开头 |
| 禁止符号 | 用了emoji/代码块 | 全文搜索禁止符号 |
| 数字格式 | 阿拉伯数字 vs 中文 | 全文替换检查 |
| 无总结词 | "总之"/"综上" | 搜索并删除总结性词语 |
| 纯净输出 | 多了解释文字 | 删除所有非要求内容 |

### 4. 正反模式

```
❌ 错误: 看到格式约束直接回答，不检查
❌ 错误: "我注意到格式要求"但实际违反了1条
❌ 错误: 约束数量多时遗漏最后1条

✅ 正确: 先数有几个格式约束 → 逐一满足 → 输出前最后检查一遍
✅ 正确: 高风险约束（如"不要用emoji"）在输出后全文搜索确认
```

## 触发条件

当问题中出现以下模式时自动触发：
- "格式要求"/"严格按照"/"不要使用"
- "不超过 N 字"/"恰好 N 句"
- "必须以 X 开头/结尾"
- "用中文数字"/"不用代码块"
- 列出 3+ 条格式规则

## 预期效果

- SE-010 类型题（格式约束）: 9→10 (+1)
- 所有格式约束问题: 格式违反率 20% → 5%
