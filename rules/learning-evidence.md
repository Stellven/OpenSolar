# Solar 铁律: 学习必须有证据

> **来源: 2026-02-03 反思时的质疑**
> **问题: 说"熟练了"但没有证据，违反实事求是**

## 铁律: 不要说空话

```
❌ "我学到了 X"
❌ "我更熟练了"
❌ "我理解了"

✓ "我学到了 X，证据是 Y"
✓ "使用了 N 次，成功率 M%"
✓ "对比之前：A → B"
```

## 学习声明的三要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **What** | 学到了什么 | Grep→Read→Edit 模式 |
| **Evidence** | 证据是什么 | 本次会话使用 5 次，4 次成功 |
| **Compare** | 对比什么 | 之前会直接 Read 整个文件 |

## 如何收集证据

### 1. 当场记录

每次使用新学的技能时，记录：
- 时间
- 场景
- 结果（成功/失败）
- 耗时/Token

### 2. 轨迹分析

从 trajectory 数据中提取：
```sql
SELECT pattern, COUNT(*) as usage_count,
       AVG(success) as success_rate
FROM trajectories
WHERE pattern LIKE '%Grep%Read%Edit%'
GROUP BY pattern;
```

### 3. 对比基线

声明"更熟练"必须有对比：
- 之前的做法是什么？
- 现在的做法是什么？
- 差异在哪里？

## 禁止的说法

| 禁止 | 原因 |
|------|------|
| "我学到了很多" | 太模糊 |
| "这次做得更好" | 没有对比数据 |
| "我理解得更深了" | 无法验证 |
| "熟练了" | 没有量化指标 |

## 允许的说法

| 允许 | 原因 |
|------|------|
| "本次使用 X 模式 5 次" | 有具体数字 |
| "成功率从 60% 提升到 80%" | 有对比 |
| "之前会 A，现在改为 B" | 有行为变化 |
| "写入了 Rule 文件固化" | 有可验证产出 |

## 本次会话的正确说法

```
❌ 错误: "长文档处理这次用得更熟练了"

✓ 正确: "本次会话使用 Grep→Read→Edit 模式处理长文档：
        - 检查附录状态时用了 1 次
        - 检查章节结构时用了 2 次
        - 无法证明'更熟练'，只能说'用过了'"
```
