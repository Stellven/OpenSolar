# Solar 铁律: 输出即固化

> **来源**: 2026-02-27 监护人明确要求
> **核心**: 对话是缓存，文件是真相。输出不能只留在对话里。

## 铁律定义

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   每次分析/总结/设计/评审完成后                             │
│          ↓                                                  │
│   在回复监护人的同时                                        │
│          ↓                                                  │
│   必须把内容存入知识库                                      │
│                                                             │
│   ❌ 禁止: 输出只留在对话中                                 │
│   ✅ 必须: 输出 = 回复 + 存储                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 触发条件 (满足任一即存储)

| 内容类型 | 特征 | 存储位置 |
|----------|------|----------|
| 分析报告 | 带有"分析"、"报告"、"总结" | sys_favorites |
| 设计方案 | 架构图、方案对比、决策 | sys_favorites |
| 评审结论 | 对比分析、优缺点、建议 | sys_favorites |
| 技术调研 | 最佳实践、技术选型 | cortex_sources |
| 规则定义 | 新规则、流程定义 | cortex_sources |
| 经验教训 | 踩坑、bug修复、教训 | cortex_sources |

## 执行方式

```sql
-- sys_favorites (重要性 ≥ 7)
INSERT INTO sys_favorites (title, question, answer, tags, importance, created_at)
VALUES ('标题', '问题/触发词', '完整内容', '["标签"]', 重要性, datetime('now'));

-- cortex_sources (知识点)
INSERT INTO cortex_sources (citation_key, title, finding, task_id, credibility, created_at)
VALUES ('引用键', '标题', '发现/结论', '任务ID', 置信度, datetime('now'));
```

## 重要性评分

| 分数 | 标准 |
|------|------|
| 9-10 | 架构设计、核心机制、铁律 |
| 7-8 | 技术方案、分析报告、评审结论 |
| 5-6 | 一般说明、操作指南 |

## 自检清单

每次输出后：

- [ ] 这是有价值的内容吗？
- [ ] 已经存入知识库了吗？
- [ ] 标签加了吗？
- [ ] 重要性评分合理吗？

## 违反后果

```
违反 = 知识只留在对话中 = compact 后丢失
违反 = 重复劳动 = 下次还要重新分析
违反 = 浪费监护人时间
```
