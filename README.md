# Solar: A Multi-Agent Development Framework

> 让 AI 自动协调多个专业 Agent 完成复杂开发任务

## 核心理念

**用户是领导，只需描述需求。** Solar 自动：
- 分析任务复杂度
- 拆解为子任务
- 分配给合适的 Agent
- 并行执行独立任务
- 汇总结果报告

## 特性

### 1. 自动调度
```
简单任务 (<50行)   → 直接执行，不调 Agent
中等任务 (50-500行) → 串行调用 1-2 个 Agent
复杂任务 (>500行)   → 并行调用 3+ 个 Agent
```

### 2. 努力程度控制
| 复杂度 | 最大 Token | 最大 Agent | 超时 |
|--------|-----------|-----------|------|
| 简单 | 5K | 0 | 2分钟 |
| 中等 | 30K | 2 | 10分钟 |
| 复杂 | 100K | 5 | 30分钟 |

### 3. 错误恢复
- 自动保存检查点
- 失败时恢复到上一个成功状态
- 支持手动回滚

### 4. Token 优化
- 精简的 Agent 定义
- 渐进式工具暴露
- 输出格式精简

## 9 个 Agent

| 层级 | Agent | 模型 | 职责 |
|------|-------|------|------|
| 决策 | Architect | Opus | 架构设计、技术评审 |
| 决策 | PM | Opus | 产品竞争力、功能验收 |
| 决策 | Secretary | Sonnet | 记录整理、Agent 评估 |
| 执行 | Coder | Sonnet | 代码实现、重构 |
| 执行 | Tester | Sonnet | 测试编写、执行 |
| 执行 | Reviewer | Sonnet | 代码审查、安全检查 |
| 支撑 | Docs | Sonnet | 文档生成、更新 |
| 支撑 | Ops | Sonnet | 构建、部署、基准测试 |
| 支撑 | Guard | Haiku | 规范检查、质量门禁 |

## 工作流程示例

**用户：** "帮我实现用户认证模块"

**Solar 自动执行：**
```
1. 判断复杂度: 复杂 (跨模块)
2. 调用 Architect 评审方案
3. 创建检查点
4. 分解子任务，逐步执行
5. 并行: Tester + Docs
6. Reviewer 审查
7. PM 验收
8. Secretary 记录
9. 报告完成
```
