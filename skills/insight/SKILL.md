# /insight - 洞察报告生成 v3.0

> 四专家互评 + 分章节持久化 + Token 优化

## 触发词

| 触发词 | 模式 | 说明 |
|--------|------|------|
| `/insight <主题>` | 快速洞察 | 对话内，单主题，3专家 |
| `洞察分析：<主题>` | 快速洞察 | 同上 |
| `深入洞察 <主题>` | 完整报告 | 多章节，4专家，分步持久化 |
| `/insight status` | - | 查看当前进度 |
| `/insight resume <id>` | - | 恢复中断的会话 |

## 模式一：快速洞察 (对话内)

**适用场景**: 论文分析、技术调研、单主题深度分析

**执行方式**: 直接在对话中执行，3专家并行分析

**输出位置**:
- 文件: `~/Solar/docs/insights/<日期>_<主题>.md`
- 数据库: `sys_favorites` (importance=9, tags=["insight", "analysis"])

## 模式二：深入洞察 (完整报告)

### 触发词: `深入洞察 <主题>`

### 核心原则

```
┌─────────────────────────────────────────────────────────────────┐
│  TOKEN 优化铁律                                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  主脑 (Claude) 只做:                                            │
│  ✅ 编排调度                                                    │
│  ✅ 发任务给牛马                                                │
│  ✅ 汇总结果文件路径                                            │
│  ✅ 最终合并（用工具，不读内容）                                │
│                                                                 │
│  主脑 ❌ 禁止:                                                  │
│  ✗ 读取大段专家输出内容                                        │
│  ✗ 在上下文中保留章节全文                                      │
│  ✗ 手动复制粘贴内容                                            │
│                                                                 │
│  牛马负责:                                                      │
│  🐂 生成内容 → 直接写入文件                                    │
│  🐂 互评内容 → 直接写入文件                                    │
│  🐂 综合内容 → 直接写入文件                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 四大老专家

| 专家 | 模型 | 角色 | 擅长 |
|------|------|------|------|
| 稳健派 | gemini-2.5-pro | 严谨审核 | 一致性、找茬 |
| 探索派 | gemini-3-pro | 创新探索 | 方案设计、权衡 |
| 创想家 | deepseek-v3 | 创意写作 | 中文表达、代码 |
| 审判官 | deepseek-r1 | 深度推理 | 本质分析、综合 |

### 八阶段流程

```
┌─────────────────────────────────────────────────────────────────┐
│              DEEP INSIGHT v3.0 - 八阶段流程                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  阶段1: 规划 (Planning)                                         │
│    └── 审判官分析主题，确定章节数和方向                         │
│    └── 输出: planning.md                                        │
│                                                                 │
│  阶段2: 大纲生成 (Outlining) - 四专家并行                       │
│    ├── 稳健派 → outline_a.md                                    │
│    ├── 探索派 → outline_b.md                                    │
│    ├── 创想家 → outline_c.md                                  │
│    └── 审判官 → outline_d.md                                    │
│                                                                 │
│  阶段3: 大纲互评 (Outline Review) - 交叉审核                    │
│    ├── 每个专家评价其他3个大纲                                  │
│    ├── 输出: outline_reviews.md                                 │
│    └── 审判官综合 → final_outline.md ✅ 持久化                  │
│                                                                 │
│  阶段4: 章节分配 (Scheduling)                                   │
│    └── 按章节分配写作任务，作者≠审核者                          │
│                                                                 │
│  阶段5: 分章写作 (Writing) - 循环执行                           │
│    对每个章节 N:                                                │
│    ├── 读取 final_outline.md 第N章大纲                          │
│    ├── 四专家并行写作:                                          │
│    │   ├── 稳健派 → ch_N_expert_a.md                            │
│    │   ├── 探索派 → ch_N_expert_b.md                            │
│    │   ├── 创想家 → ch_N_expert_c.md                          │
│    │   └── 审判官 → ch_N_expert_d.md                            │
│    ├── 持久化检查点 ✅                                          │
│    │                                                            │
│  阶段6: 章节互评 (Chapter Review)                               │
│    对每个章节 N:                                                │
│    ├── 交叉审核（作者不审自己）                                 │
│    ├── 输出: ch_N_reviews.md                                    │
│    └── 审判官综合 → ch_N_final.md ✅ 持久化                     │
│                                                                 │
│  阶段7: 合并 (Merging)                                          │
│    └── 读取所有 ch_*_final.md → final_report.md                 │
│    └── 生成执行摘要                                             │
│                                                                 │
│  阶段8: 结束 (Closing) - 绩效评估                               │
│    └── 记录每个牛马的表现到 collab_performance                  │
│    └── 输出最终报告路径                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

├── ch_1_expert_c.md               # 阶段5: 第1章-创想家

### 专家提示词模板

#### 大纲生成提示词
```
你是 {专家昵称}，性格参数 O={O} C={C} E={E} A={A} N={N}。

任务：为主题「{主题}」生成报告大纲

要求：
1. 根据你的性格特点来组织结构
2. 明确每章的核心问题和预期产出
3. 注明你认为的重点和难点

输出格式：
# 报告大纲
## 第一章：xxx
- 核心问题：
- 预期产出：
- 难点：
...
```

#### 章节写作提示词
```
你是 {专家昵称}，性格参数 O={O} C={C} E={A} A={A} N={N}。

任务：撰写第 {N} 章「{章节标题}」

章节大纲：
{从 final_outline.md 提取的第N章内容}

要求：
1. 遵循你的性格特点和写作风格
2. 深入分析，有理有据
3. 包含你独特的见解
4. 使用 Markdown 格式

直接输出章节内容，开头写 # 第{N}章：{章节标题}
```

#### 互评提示词
```
你是 {专家昵称}，现在作为评审者。

请评价以下内容（来自其他专家的第 {N} 章）：
{章节内容}

评价维度：
1. 内容完整性 (1-10)
2. 逻辑清晰度 (1-10)
3. 深度和见解 (1-10)
4. 表达质量 (1-10)

请指出：
- 优点（至少2条）
- 不足（至少2条）
- 改进建议

禁止：互相吹捧。必须指出真实问题。
```

### 防作弊机制

| 机制 | 说明 |
|------|------|
| 匿名化 | 互评时不透露作者身份 |
| 交叉审核 | 作者不审自己的内容 |
| 挑战者 | 稳健派专门找茬 |
| 结构化清单 | 必须逐项评分 |
| 问责制 | 审核者对评分负责 |

### 执行方式

#### 方式一：CLI 直接运行 (推荐)

```bash
# 独立进程执行，不受上下文限制
bun ~/.claude/core/solar-farm/insight-agent-v3.ts "主题" <章节数>

# 示例
bun ~/.claude/core/solar-farm/insight-agent-v3.ts "AI Agent 记忆系统" 4

       [阶段1] 规划中...
       [阶段2] 四专家生成大纲中... (并行)
       ...
```

### Token 消耗估算

| 阶段 | 专家调用次数 | 预估 Token | 说明 |
|------|-------------|-----------|------|
| 规划 | 1 | 2K | 审判官 |
| 大纲生成 | 4 | 8K | 四专家并行 |
| 大纲互评 | 4 | 6K | 交叉审核 |
| 分章写作 | 4×N | 8K×N | N章×4专家 |
| 章节互评 | 4×N | 6K×N | N章×4专家 |
| 合并综合 | 1 | 4K | 审判官 |
| **总计** | - | ~20K+14K×N | N=4 时约 76K |

**关键优化**: 主脑不读取内容，只传递文件路径 → 主脑 Token 消耗 <5K
