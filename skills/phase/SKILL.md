# /phase - 阶段转换

## 命令

| 命令 | 说明 |
|------|------|
| `/phase next` | 转换到下一阶段 |
| `/phase P1-P5` | 跳转到指定阶段 (跳过Gate) |

## 转换流程

1. 读取 `.solar/flow-state.json`
2. 执行 Gate 检查 (如需要)
3. 更新状态: `{"phase":"<新>","agent":"<新>"}`
4. 输出转换信息 + 新 Agent 宣告

## Gate 检查

| Gate | 触发 | 要求 | 失败 |
|------|------|------|------|
| G1 | P2→P3 | 设计文档存在 | 重试2次 |
| G2 | P4→P5 | 测试通过 | 重试3次 |

跳转命令 `/phase P3` 跳过 Gate (显示警告)

## 阶段详情

| 阶段 | Agent | 产出 | 限制 |
|------|-------|------|------|
| P1 | 🔬 Researcher | 研究报告 | 禁止 Edit/Bash |
| P2 | 🏗️ Architect | 设计文档 | 只读命令 |
| P3 | 💻 Coder | 代码 | 无 |
| P4 | 🧪 Tester | 测试报告 | 无 |
| P5 | ⚙️ Ops | 部署/文档 | 无 |

## 输出格式

```
┌─ ☀️ Solar 阶段转换 ────────────────────┐
│ P2 Architect → P3 Coder               │
│ Agent: 💻 Coder | 职责: 代码实现       │
└───────────────────────────────────────┘

┌─ 💻 Coder ────────────────────────────┐
│ Task: <任务>                          │
│ Plan: 1. <步骤1>  2. <步骤2>         │
└───────────────────────────────────────┘
```
