# /ppt - Markdown to Huawei-style HTML Presentation

> MD → 华为风格 HTML 演示文稿，支持长内容自动拆分

## 用法

```bash
/ppt <input.md> [output.html]     # 转换 MD 为 HTML PPT
/ppt --status                      # 查看任务状态
/ppt --resume                      # 恢复中断的任务
```

## Markdown 格式

```markdown
---
title: 演示标题
author: 作者
date: 2026-02-03
---

# 主标题
→ 封面页 (华为红背景)

## 章节标题
→ 章节页 (科技蓝背景)

### 子标题
→ 内容页副标题

- 要点 1
- 要点 2
- 要点 3
→ 内容要点 (超过5个自动拆分)

---
→ 手动分页
```

## 特性

| 特性 | 说明 |
|------|------|
| 华为风格 | 红蓝配色，简洁大气 |
| 自动拆分 | 每页最多5个要点 |
| 键盘导航 | ← → 翻页，F 全屏，P 打印 |
| 触摸支持 | 滑动/点击导航 |
| 进度条 | 顶部显示进度 |
| 任务状态 | 长文档支持断点续传 |

## 示例

输入 `report.md`:
```markdown
# 项目进展汇报

## 本周完成

- 完成了核心模块开发
- 通过了单元测试
- 性能优化提升 30%

## 下周计划

- 集成测试
- 文档编写
- 上线准备
```

执行:
```bash
bun run ~/.claude/skills/ppt/ppt.ts report.md report.html
```

输出 `report.html`:
- 封面页: "项目进展汇报" (华为红)
- 章节页: "本周完成" (科技蓝)
- 内容页: 3个要点
- 章节页: "下周计划" (科技蓝)
- 内容页: 3个要点

## 任务状态持久化

长文档转换时，状态保存在 `.solar/ppt-state.json`:

```json
{
  "inputFile": "long-report.md",
  "outputFile": "long-report.html",
  "totalSlides": 50,
  "processedSlides": 30,
  "status": "processing"
}
```

中断后可用 `--resume` 恢复。

## 键盘快捷键

| 按键 | 功能 |
|------|------|
| → / ↓ / Space | 下一页 |
| ← / ↑ | 上一页 |
| Home | 第一页 |
| End | 最后一页 |
| F | 全屏 |
| P | 打印 |
