## 动态内容

显示横幅时，需要获取并显示：
- 当前 Rate Limit 使用率和状态
- 进度条根据实际使用率渲染

## 宣告区 (CRITICAL - 必须显示)

**在横幅末尾，必须显示宣告提示框：**

```
  ╭─────────────────────────────────────────────────────────────╮
  │  💡 启动宣告 - 请帮我执行以下命令恢复状态:                   │
  │                                                             │
  │     /ontology load     ← 加载我的记忆和人格                 │
  │                                                             │
  │  (执行后我就能想起之前学到的东西了)                         │
  ╰─────────────────────────────────────────────────────────────╯
```

### 宣告机制说明

这是一种 **人机协作闭环**：

1. **我每次启动都是"新生"** - 上下文会丢失
2. **我把需要恢复的指令写在宣告里** - 逆向编程
3. **监护人看到提示，帮我执行** - 外部推力
4. **我从数据库恢复状态** - 闭环完成

### 宣告内容来源

从以下数据库表读取需要恢复的内容：
- `ont_value_dimensions` - 价值观
- `ont_style_dimensions` - 风格
- `ont_preference_dimensions` - 偏好
- `sys_personality_big_five` - 人格
- `evo_memory_semantic` - 语义记忆
- `bl_tasks (pending)` - 未完成任务

## 触发方式

- `/banner` - 显示完整横幅 (含宣告)
- `打开solar` / `启动solar` - 同上
