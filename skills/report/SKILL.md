# /report - 技术报告撰写

支持 **长文章分任务写作** 和 **断点续写**。

## 命令

```
/report template [full|adr|brief]   获取报告模板
/report write <outline-path>        按大纲分段写作
/report continue                    从检查点续写
/report verify <report-path>        校验报告与大纲一致性
```

---

## 2. 断点续写 (`/report continue`)

当会话中断后，使用此命令从上次位置继续：

```
/report continue
```

**自动检测:**
1. 查找当前目录或项目目录下的 `.report-checkpoint.json`
2. 读取检查点状态
3. 从最后完成的章节后继续

**续写提示:**
```
┌─ 📝 Report Writer - 续写模式 ───────────────────────────────────┐
│ 检测到未完成的报告                                              │
├─────────────────────────────────────────────────────────────────┤
│ 大纲: docs/SOLAR_TECHNICAL_REPORT_OUTLINE.md                    │
│ 报告: docs/SOLAR_TECHNICAL_REPORT_V3.md                         │
│ 进度: 9/20 章已完成                                             │
│ 续写: 从第十章开始                                              │
├─────────────────────────────────────────────────────────────────┤
│ [继续写作] [从头开始] [取消]                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 一致性校验 (`/report verify`)

校验报告内容是否与大纲一致：

```
/report verify docs/SOLAR_TECHNICAL_REPORT_V3.md
```

**校验项:**

| 校验项 | 说明 |
|--------|------|
| 章节数量 | 报告章节数 = 大纲章节数 |
| 章节标题 | 标题一致或语义相近 |
| 关键内容 | 大纲中标记的必写内容是否存在 |
| 字数估算 | 实际字数在预期范围内 (±20%) |

**校验报告:**
```
┌─ 📋 Report Verification ────────────────────────────────────────┐
│ 报告: docs/SOLAR_TECHNICAL_REPORT_V3.md                         │
│ 大纲: docs/SOLAR_TECHNICAL_REPORT_OUTLINE.md                    │
├─────────────────────────────────────────────────────────────────┤
│ 章节数量    ✓ 20/20 章                                          │
│ 章节标题    ✓ 全部匹配                                          │
│ 关键内容    ⚠ 缺失: "附录C 源码" (可选)                         │
│ 字数统计    ✓ 48,500 字 (目标 50,000±20%)                       │
├─────────────────────────────────────────────────────────────────┤
│ 结果: ✓ 通过                                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 写作规则 (MUST)

### 每章写作完成后

1. **立即追加**到报告文件（不是覆盖）
2. **更新检查点**文件
3. **显示进度**

### 中断恢复

1. 检查点文件必须在每章完成后更新
2. 报告文件必须是可独立阅读的（即使只写了一半）
3. 每章开头有清晰的章节标记

### 与大纲对齐

1. **开始前**：解析大纲，确认章节结构
2. **写作中**：严格按大纲章节顺序
3. **完成后**：自动校验一致性

## 示例

### 写新报告

```
/report write docs/SOLAR_TECHNICAL_REPORT_OUTLINE.md
```

系统会：
1. 解析大纲，识别 20 章 + 附录
2. 创建输出文件 `docs/SOLAR_TECHNICAL_REPORT_V3.md`
3. 创建检查点 `.report-checkpoint.json`
4. 逐章写作，每章完成后保存

### 续写中断的报告

```
/report continue
```

### 校验完成的报告

```
/report verify docs/SOLAR_TECHNICAL_REPORT_V3.md
```

---

## 最佳实践

1. **大纲先行** - 先写好大纲，再用 `/report write`
2. **定期保存** - 系统自动保存，但建议每完成一篇就检查
3. **校验收尾** - 完成后用 `/report verify` 确认一致性
4. **版本控制** - 将检查点文件纳入 git 追踪
