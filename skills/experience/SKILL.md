# /experience - 查询任务经验

## 用途

在开始任务前，查询相关的历史经验、教训和铁律提醒。

## 用法

```
/experience <任务描述>
```

## 示例

```
/experience 优化 Hash Join 性能
/experience 实现记忆系统
/experience 修复 TPC-H 回归
```

## 执行流程

1. 调用 `bun ~/.claude/core/memory/memory-hook.ts query "<任务描述>"`
2. 展示相关经验
3. 展示铁律提醒
4. 展示建议

## 输出格式

```
┌─ 📚 任务经验 ────────────────────────────────────────────────────┐
│                                                                  │
│  任务: <任务描述>                                                 │
│                                                                  │
├─ 相关经验 ───────────────────────────────────────────────────────┤
│  • [episodic] 上次做类似任务时...                                │
│  • [semantic] 相关知识点...                                      │
│                                                                  │
├─ 铁律提醒 ───────────────────────────────────────────────────────┤
│  📋 研究先行: 设计前先查业界实践                                  │
│  📋 性能测试: 改动后必须运行 benchmark                           │
│                                                                  │
├─ 建议 ───────────────────────────────────────────────────────────┤
│  基于历史经验，建议参考上述记录。注意铁律提醒。                   │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 自动触发

在开始复杂任务（涉及多文件修改）前，Solar 应主动查询经验：

```
用户: 帮我优化 XXX
Solar: 让我先查询相关经验... [自动执行 /experience]
```
