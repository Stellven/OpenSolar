# /reflect - 元认知反思

> 基于智慧法则的结构化反思

## 用法

```bash
/reflect                # 对当前会话进行反思
/reflect learn <topic>  # 记录学习要点
/reflect review         # 查看历史反思
```

## 反思框架 (必须遵循)

每次反思必须经过以下检验：

### 1. 实事求是 - 发生了什么？

```
观察客观事实，不加主观臆断：
- 完成了什么任务？
- 遇到了什么问题？
- 产生了什么结果？
```

### 2. 辩证分析 - 为什么会这样？

```
分析因果关系和对立统一：
- 成功的原因是什么？
- 失败的原因是什么？
- 有哪些矛盾需要平衡？
```

### 3. 经典参照 - 符合哪条法则？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  法则            检视问题                                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│  知行合一        我学到的东西用了吗？验证了吗？                              │
│  实事求是        我的判断基于客观事实吗？有没有自欺？                        │
│  中庸之道        我是否走极端了？有没有更平衡的方案？                        │
│  否定之否定      这次迭代比上次更好吗？螺旋上升了吗？                        │
│  实践检验        这个结论经过实践验证了吗？                                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4. 假设生成 - 如果改变 X？

```
基于分析，提出改进假设：
- 假设: 如果改变 X，可能会...
- 预期: 结果应该是...
- 验证方法: 下次遇到类似情况时...
```

### 5. 知识固化 - 存入还是放弃？

```
判断是否值得记住：
- 置信度: 高/中/低
- 适用范围: 通用/特定场景
- 行动: 存入记忆 / 标记待验证 / 放弃
```

## 自动触发

在以下情况自动触发反思：

1. **会话结束前** - SessionStop hook
2. **重大任务完成后** - 手动调用
3. **遇到错误/失败时** - 分析原因

## 数据存储

```sql
-- 反思记录表
CREATE TABLE IF NOT EXISTS sys_reflections (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT DEFAULT (datetime('now', 'localtime')),
    session_id TEXT,
    summary TEXT,
    wisdom_checks TEXT,     -- JSON: {"知行合一": true, ...}
    learnings TEXT,         -- JSON: [{point, confidence}, ...]
    action_items TEXT,      -- JSON: [...]
    stored_to_memory INTEGER DEFAULT 0
);
```
