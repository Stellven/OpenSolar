# Solar Data Agent 设计文档

> **愿景**: 构建一个类 Palantir 的本体论驱动数据智能体，融合所有可访问数据源，形成统一的知识图谱，指导人机协作决策。

### 1.2 Palantir 核心启发

基于 [Palantir Ontology](https://www.palantir.com/platforms/ontology/) 的三层架构:

| 层级 | Palantir | Solar 对应 | 说明 |
|------|----------|------------|------|
| **Semantic** | Object Types, Properties, Links | 实体定义、属性、关系 | 语义模型 |
| **Kinetic** | Functions, Actions | MCP 工具、Skills、脚本 | 操作能力 |
| **Dynamic** | Security, Permissions | 监护人确认、访问控制 | 安全治理 |

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **IaST (Infrastructure as Tables)** | 一切皆系统表，可查询可演化 |
| **Ontology-First** | 先建模后开发，本体论驱动 |
| **Human-in-the-Loop** | 重要决策需监护人确认 |
| **Incremental Evolution** | 渐进式演进，从核心实体开始 |
| **Privacy-Aware** | 敏感数据本地处理，不外传 |

### 3.2 实体定义

```sql
-- 核心实体: Person (人)
CREATE TABLE ont_person (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    role TEXT,                    -- 'guardian', 'collaborator', 'contact'
    source TEXT,                  -- 'wechat', 'email', 'github', etc.
    properties JSONB,             -- 扩展属性
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME
);

-- 核心实体: Project (项目)
CREATE TABLE ont_project (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    path TEXT,                    -- 本地路径
    repo_url TEXT,                -- Git 仓库
    tech_stack JSONB,             -- 技术栈
    status TEXT,                  -- 'active', 'archived', 'planning'
    metrics JSONB,                -- 性能指标、代码统计
    created_at DATETIME,
    updated_at DATETIME
);

-- 核心实体: Session (会话)
CREATE TABLE ont_session (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES ont_project(id),
    summary TEXT,
    message_count INTEGER,
    token_usage JSONB,
    tools_used JSONB,
    outcome TEXT,                 -- 'success', 'partial', 'failed'
    insights JSONB,               -- 提取的洞察
    start_time DATETIME,
    end_time DATETIME
);

-- 核心实体: Memory (记忆)
CREATE TABLE ont_memory (
    id TEXT PRIMARY KEY,
    type TEXT,                    -- 'episodic', 'semantic', 'procedural'
    content TEXT,
    embedding BLOB,               -- 向量嵌入
    source_type TEXT,             -- 'session', 'document', 'insight'
    source_id TEXT,
    importance REAL,              -- 0-1 重要度
    decay_rate REAL,              -- 衰减率
    last_accessed DATETIME,
    created_at DATETIME
);

-- 工作实体: Task (任务)
CREATE TABLE ont_task (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT,
    project_id TEXT REFERENCES ont_project(id),
    assignee TEXT,                -- 'human', 'solar', 'agent:xxx'
    status TEXT,                  -- 'pending', 'in_progress', 'completed', 'blocked'
    priority TEXT,                -- 'P0', 'P1', 'P2', 'P3'
    due_date DATETIME,
    completed_at DATETIME,
    created_at DATETIME
);

-- 工作实体: Code (代码)
CREATE TABLE ont_code (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES ont_project(id),
    file_path TEXT,
    language TEXT,
    type TEXT,                    -- 'function', 'class', 'module', 'file'
    name TEXT,
    signature TEXT,
    doc_string TEXT,
    metrics JSONB,                -- 复杂度、行数等
    embedding BLOB,
    updated_at DATETIME
);

-- 工作实体: Insight (洞察)
CREATE TABLE ont_insight (
    id TEXT PRIMARY KEY,
    type TEXT,                    -- 'pattern', 'anomaly', 'opportunity', 'risk'
    title TEXT,
    description TEXT,
    evidence JSONB,               -- 支撑证据
    confidence REAL,              -- 置信度
    impact TEXT,                  -- 'high', 'medium', 'low'
    status TEXT,                  -- 'new', 'acknowledged', 'acted', 'dismissed'
    source_ids JSONB,             -- 来源实体
    created_at DATETIME
);

-- 辅助实体: Document (文档)
CREATE TABLE ont_document (
    id TEXT PRIMARY KEY,
    title TEXT,
    type TEXT,                    -- 'design', 'report', 'note', 'paper'
    source TEXT,                  -- 'local', 'notion', 'arxiv', 'web'
    content TEXT,
    summary TEXT,
    embedding BLOB,
    url TEXT,
    created_at DATETIME,
    updated_at DATETIME
);

-- 辅助实体: Metric (指标)
CREATE TABLE ont_metric (
    id TEXT PRIMARY KEY,
    name TEXT,
    entity_type TEXT,             -- 关联的实体类型
    entity_id TEXT,               -- 关联的实体 ID
    value REAL,
    unit TEXT,
    timestamp DATETIME,
    metadata JSONB
);

-- 辅助实体: Decision (决策)
CREATE TABLE ont_decision (
    id TEXT PRIMARY KEY,
    title TEXT,
    description TEXT,
    options JSONB,                -- 选项列表
    chosen_option TEXT,
    rationale TEXT,               -- 决策理由
    made_by TEXT,                 -- 'guardian', 'solar', 'auto'
    related_entities JSONB,       -- 相关实体
    outcome TEXT,                 -- 后续评估
    created_at DATETIME
);
```

### 3.3 关系定义 (Links)

```sql
-- 通用关系表
CREATE TABLE ont_links (
    id TEXT PRIMARY KEY,
    source_type TEXT NOT NULL,
    source_id TEXT NOT NULL,
    link_type TEXT NOT NULL,      -- 关系类型
    target_type TEXT NOT NULL,
    target_id TEXT NOT NULL,
    properties JSONB,             -- 关系属性
    weight REAL DEFAULT 1.0,      -- 关系权重
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 关系类型索引
CREATE INDEX idx_links_source ON ont_links(source_type, source_id);
CREATE INDEX idx_links_target ON ont_links(target_type, target_id);
CREATE INDEX idx_links_type ON ont_links(link_type);
```

**核心关系类型:**

| 关系类型 | 源 → 目标 | 说明 |
|----------|-----------|------|
| `owns` | Person → Project | 人拥有项目 |
| `participates_in` | Person → Session | 人参与会话 |
| `belongs_to` | Session → Project | 会话属于项目 |
| `produces` | Session → Insight | 会话产生洞察 |
| `references` | Memory → * | 记忆引用任何实体 |
| `depends_on` | Task → Task | 任务依赖 |
| `implements` | Code → Task | 代码实现任务 |
| `mentions` | Document → * | 文档提及实体 |
| `derived_from` | Insight → Metric | 洞察来自指标 |
| `influences` | Decision → * | 决策影响实体 |

### 4.2 组件详解

#### 4.2.1 智能层 Agents

| Agent | 职责 | 触发方式 |
|-------|------|----------|
| **Query Agent** | 自然语言查询本体论 | 用户提问 |
| **Insight Agent** | 发现模式、检测异常、生成洞察 | 定时/数据变更 |
| **Recommendation Agent** | 行动建议、优先级排序 | 洞察触发/请求 |
| **Memory Agent** | 记忆管理、关联发现、知识迁移 | 会话结束 |
| **Sync Agent** | 增量同步各数据源 | 定时/手动 |
| **Evolution Agent** | 本体 Schema 演进、实体类型发现 | 数据变更 |

interface ConnectorConfig {

## 第八章: 与 Palantir 对比

| 维度 | Palantir | Solar Data Agent | 说明 |
|------|----------|------------------|------|
| **规模** | 企业级 TB-PB | 个人级 GB | 轻量化 |
| **部署** | 云/私有云 | 本地优先 | 隐私保护 |
| **Schema** | 预定义 + 动态 | 预定义 + 演进 | 渐进式 |
| **安全** | RBAC 复杂权限 | 监护人确认 | 简化模型 |
| **AI** | AIP Agent Studio | Claude Code Agents | LLM 驱动 |
| **成本** | $$$$$$ | Token 成本 | 极低 |

**核心差异:**
- Palantir 面向企业大规模数据融合
- Solar Data Agent 面向个人/小团队知识管理
- 共同理念: **本体论驱动 + AI 增强**
