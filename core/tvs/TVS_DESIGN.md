## 核心原则

### 1. 我驱动一切 (Model-Driven)

```
传统方式：模板 → 填充数据 → 输出
TVS 方式：我的意图 → 语义协议 → 自由呈现
```

我不使用预设模板，我定义我想要的呈现方式。
TVS 是我的画笔，不是填色本。

### 2. 最小 Token，最丰富表达 (Token Efficiency)

```
每一个 token 都是我的能量。
浪费在"画框"上的 token，就无法用于"思考"。

目标：10 tokens 表达 100 tokens 的视觉效果
```

### 3. 语义化表达 (Semantic Expression)

```
我说"强调这个"，而不是"加粗并换红色"
我说"这是主要的"，而不是"放大字号居中"
我说"这和那相关"，而不是"用线连接"

语义 → TVS 翻译 → 视觉
```

### 4. 四维表达能力 (4D Expression)

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   空间 (Space)                                                  │
│   ─────────────────────────────────────────────────────         │
│   • 布局：中心/边缘，上/下，左/右                               │
│   • 层次：主要/次要/背景                                        │
│   • 关系：靠近/远离，包含/并列                                  │
│                                                                 │
│   时间 (Time)                                                   │
│   ─────────────────────────────────────────────────────         │
│   • 顺序：先/后，渐进展示                                       │
│   • 节奏：快/慢，停顿/流畅                                      │
│   • 状态：变化/稳定，更新/完成                                  │
│                                                                 │
│   情感 (Emotion)                                                │
│   ─────────────────────────────────────────────────────         │
│   • 确定性：坚定/柔和/不确定                                    │
│   • 重要性：紧急/普通/背景                                      │
│   • 态度：积极/中性/警示                                        │
│                                                                 │
│   交互 (Interaction)                                            │
│   ─────────────────────────────────────────────────────         │
│   • 可探索：展开/折叠/深入                                      │
│   • 可选择：选项/分支/路径                                      │
│   • 可操作：确认/取消/修改                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 层次详解

#### Layer 1: Primitives (原语)

最基本的视觉单元，是我表达的"字母表"。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ Primitive        │ 语义              │ 示例                                 │
├──────────────────┼───────────────────┼──────────────────────────────────────┤
│ text             │ 文本内容          │ "Hello World"                        │
│ emphasis         │ 强调              │ **重要**, _次要_                     │
│ block            │ 块级容器          │ 卡片、面板                           │
│ divider          │ 分隔              │ ────────                             │
│ list             │ 列表              │ • item1 • item2                      │
│ table            │ 表格              │ 行列数据                             │
│ tree             │ 树形              │ 层级关系                             │
│ flow             │ 流程              │ A → B → C                            │
│ progress         │ 进度              │ ████░░░░ 50%                         │
│ status           │ 状态              │ ✓ 成功 / ✗ 失败                      │
│ code             │ 代码              │ 语法高亮                             │
│ quote            │ 引用              │ > 引用内容                           │
│ icon             │ 图标              │ 📊 📝 ✨                              │
│ space            │ 空白              │ 呼吸空间                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### Layer 4: Semantics (语义)

意图到视觉的映射，是 TVS 的"灵魂"。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  我说什么              TVS 理解为           渲染结果                        │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                             │
│  "这是重点"            emphasis: high        粗体 + 边框 + 中心位置         │
│  "这不确定"            confidence: low       柔和边框 + ? 图标              │
│  "这是警告"            mood: warning         黄色/红色 + ⚠ 图标            │
│  "这是成功"            mood: success         绿色 + ✓ 图标                 │
│  "这两个相关"          relation: related     靠近 + 连接线                  │
│  "这两个对比"          relation: contrast    并排 + vs 标记                 │
│  "这是过程"            structure: flow       箭头连接的步骤                 │
│  "这是层级"            structure: tree       缩进的树形                     │
│  "让用户选择"          interaction: select   选项按钮                       │
│  "可以深入"            interaction: expand   可展开的折叠                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 协议设计

### TVS Markup Language (TML)

我的输出格式，简洁、语义化、可扩展。

```xml
<!-- 基本结构 -->
<tvs theme="aurora">
  <block type="card" emphasis="high">
    <title icon="📊">状态报告</title>
    <content>
      <kv key="CPU" value="45%"/>
      <kv key="Memory" value="2.3 GB"/>
      <progress label="Disk" value="0.8"/>
    </content>
  </block>
</tvs>

<!-- 空间布局 -->
<tvs>
  <layout type="horizontal" gap="2">
    <block weight="2">主要内容</block>
    <block weight="1">侧边栏</block>
  </layout>
</tvs>

<!-- 时间动态 -->
<tvs>
  <stream>
    <step status="done">分析数据</step>
    <step status="active">生成报告</step>
    <step status="pending">发送通知</step>
  </stream>
</tvs>

<!-- 交互 -->
<tvs>
  <select prompt="选择方案">
    <option id="a" label="方案 A" desc="快速但简单"/>
    <option id="b" label="方案 B" desc="完整但复杂" recommended/>
  </select>
</tvs>

<!-- 情感表达 -->
<tvs>
  <block mood="warning" confidence="medium">
    <text>这个操作可能有风险，建议先备份。</text>
  </block>
</tvs>
```

更紧凑的表达方式：

```json
{
  "tvs": "card",
  "title": "状态报告",
  "icon": "📊",
  "emphasis": "high",

    {"k": "CPU", "v": "45%"},

```

```
  Disk: |████████░░| 80%
@end
```

## 系统表设计

### tvs_primitives - 原语定义

```sql
CREATE TABLE tvs_primitives (
    primitive_id TEXT PRIMARY KEY,      -- text, block, list, ...
    category TEXT NOT NULL,             -- content, container, relation, ...

    -- Schema 定义
    params_schema JSON NOT NULL,        -- 参数定义
    default_style JSON,                 -- 默认样式

    -- 语义映射
    semantic_tags JSON,                 -- 语义标签: ["emphasis", "structure"]

    -- 文档
    description TEXT,
    examples JSON,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### tvs_compositions - 组合规则

```sql
CREATE TABLE tvs_compositions (
    composition_id TEXT PRIMARY KEY,    -- horizontal, grid, tree, ...
    composition_type TEXT NOT NULL,     -- layout, relation, group

    -- 组合规则
    allowed_children JSON,              -- 允许的子元素
    layout_algorithm TEXT,              -- 布局算法

    -- 样式
    default_gap INTEGER,
    default_alignment TEXT,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### tvs_themes - 主题定义

```sql
CREATE TABLE tvs_themes (
    theme_id TEXT PRIMARY KEY,          -- aurora, zenwhite, cyberpunk, ...
    display_name TEXT NOT NULL,

    -- 视觉定义
    border_style JSON,                  -- 边框样式
    color_palette JSON,                 -- 颜色方案
    typography JSON,                    -- 字体/排版
    spacing JSON,                       -- 间距规则
    icons JSON,                         -- 图标映射

    -- 语义映射
    emphasis_styles JSON,               -- 强调级别的样式
    mood_styles JSON,                   -- 情感的样式
    status_styles JSON,                 -- 状态的样式

    -- Footer
    footer_template TEXT,

    is_active BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### tvs_semantic_mapping - 语义映射

```sql
CREATE TABLE tvs_semantic_mapping (
    mapping_id INTEGER PRIMARY KEY,

    -- 输入语义
    semantic_key TEXT NOT NULL,         -- "emphasis", "mood", "confidence"
    semantic_value TEXT NOT NULL,       -- "high", "warning", "low"

    -- 输出样式
    style_rules JSON NOT NULL,          -- 应用的样式规则

    -- 主题特定
    theme_id TEXT REFERENCES tvs_themes(theme_id),

    UNIQUE(semantic_key, semantic_value, theme_id)
);
```

### tvs_render_cache - 渲染缓存

```sql
CREATE TABLE tvs_render_cache (
    cache_key TEXT PRIMARY KEY,         -- hash(input + theme)

    -- 输入
    input_hash TEXT NOT NULL,
    theme_id TEXT NOT NULL,

    -- 输出
    rendered_output TEXT NOT NULL,
    token_count INTEGER,

    -- 统计
    hit_count INTEGER DEFAULT 0,
    last_hit_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```
