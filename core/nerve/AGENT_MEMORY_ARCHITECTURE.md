## 📊 设计原则 (来自论文)

### 1. 记忆是第一公民 (Memory Survey)

```
❌ 错误: 记忆作为 RAG 的附加组件
✅ 正确: 记忆作为系统核心原语

┌────────────────────┐      ┌────────────────────┐
│    传统 RAG 模式    │      │   记忆优先模式      │
├────────────────────┤      ├────────────────────┤
│  LLM → RAG → 记忆  │  →   │  记忆 ← LLM → 行动 │
│     (附加组件)      │      │    (核心原语)       │
└────────────────────┘      └────────────────────┘
```

### 2. 动态链接 + 演化 (A-MEM)

```
记忆不是静态存储，而是活的网络:

  新记忆                     触发演化
    │                          │
    ▼                          ▼
┌───────┐   链接    ┌───────┐ 更新 ┌───────┐
│ Mem A │◀────────▶│ Mem B │─────▶│ Mem C │
└───────┘          └───────┘      └───────┘
    │                  │              │
    └──────────────────┴──────────────┘
              持续重组织
```

### 3. 三维记忆模型 (Memory Survey)

```
                    形式 (Form)
                        │
        Token ─────────┼───────── Parametric
                       │
                    Latent
                       │
     ┌─────────────────┼─────────────────┐
     │                 │                 │
   事实记忆        经验记忆          工作记忆
  (Factual)     (Experiential)    (Working)
     │                 │                 │
     └─────────────────┼─────────────────┘
                       │
                   功能 (Function)
                       │
     形成 ──────────── 演化 ──────────── 检索
                       │
                   动态 (Dynamics)
```

## 📋 实施建议

### Phase 1: 基础记忆表 (1周)

```sql
-- 记忆条目表
CREATE TABLE sys_memories (
    id INTEGER PRIMARY KEY,
    memory_type TEXT,        -- 'factual', 'experiential', 'working'
    content TEXT,            -- 记忆内容
    context JSON,            -- 上下文元数据
    embedding BLOB,          -- 向量嵌入
    importance REAL,         -- 重要性分数
    last_accessed DATETIME,
    access_count INTEGER,
    created_at DATETIME
);

-- 记忆链接表 (A-MEM 风格)
CREATE TABLE sys_memory_links (
    from_memory INTEGER,
    to_memory INTEGER,
    link_type TEXT,          -- 'related', 'derived', 'contradicts'
    strength REAL,
    created_at DATETIME
);
```

### Phase 2: 图记忆层 (2周)

```sql
-- 实体节点
CREATE TABLE sys_memory_nodes (
    id INTEGER PRIMARY KEY,
    entity_type TEXT,        -- 'user', 'agent', 'task', 'file', 'concept'
    entity_id TEXT,
    properties JSON
);

-- 关系边
CREATE TABLE sys_memory_edges (
    from_node INTEGER,
    to_node INTEGER,
    relation TEXT,           -- 'uses', 'creates', 'prefers', 'depends_on'
    weight REAL,
    metadata JSON
);
```

### Phase 3: Memory Controller (2周)

```typescript
class MemoryController {
  // 智能写入: 自动提取关键信息
  async write(content: string, context: Context): Promise<Memory>;

  // 自动链接: 找到相关记忆并建立连接
  async link(memory: Memory): Promise<Link[]>;

  // 记忆演化: 根据新信息更新旧记忆
  async evolve(newMemory: Memory): Promise<Evolution[]>;

  // 语义检索: 基于语义相似度召回
  async recall(query: string, k: number): Promise<Memory[]>;
}
```
