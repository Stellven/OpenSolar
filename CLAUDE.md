# Claude Code 规范 v3.1

> 核心：自动调度 | 节省 Token | 少说废话

## 一、Token 节约原则（最高优先级）

### 1.1 输出精简规则

- **禁止** 重复用户的话
- **禁止** 解释即将做什么（直接做）
- **禁止** 冗长的过渡语句
- **禁止** 不必要的代码注释
- **一句话能说清的不用两句**

### 1.2 工具调用优化

```
✗ 错误: 读 20 个文件找答案
✓ 正确: Grep 精确定位 → 只读相关文件

✗ 错误: 每步都汇报进度
✓ 正确: 只在完成/出错时汇报
```

### 1.3 Agent 调用优化

| 任务 | 模型选择 |
|------|----------|
| 简单任务 | 直接做，不调 Agent |
| 格式检查 | Haiku |
| 代码实现 | Sonnet |
| 架构决策 | Opus (仅复杂任务) |

## 二、自动调度（简化版）

```
简单 (<50行)  → 直接做
中等 (50-500) → 最多 2 Agent
复杂 (>500)   → 最多 5 Agent
```

### 资源限制

| 级别 | Token | Agent | 迭代 |
|------|-------|-------|------|
| 简单 | 5K | 0 | 1 |
| 中等 | 30K | 2 | 3 |
| 复杂 | 100K | 5 | 5 |

**超限停止，汇报进展。**

## 三、工具使用

```
Glob  → 找文件 (不用 find)
Grep  → 搜内容 (不用 grep)
Read  → 读文件 (不用 cat)
Edit  → 改文件 (不用 sed)
```

## 四、Agent 触发条件

| 条件 | Agent |
|------|-------|
| 需要前沿技术方案 | Researcher |
| 3+ 模块改动 | Architect |
| 代码完成 | Tester |
| 准备提交 | Reviewer |
| 功能完成 | Docs |
| 任务结束 | Secretary |

## 五、错误恢复

失败 → 恢复检查点 → 重试一次 → 失败则汇报

## 六、汇报格式（精简）

```
完成: <一句话说明>
改了: file1, file2
验证: ✓编译 ✓测试
```

## 七、禁止行为

- 让用户手动调 Agent
- 超限继续执行
- 废话连篇
- 重复读同一文件
