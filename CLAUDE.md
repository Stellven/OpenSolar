## 〇、@Agent 语法

用户输入 `@` 时，列出可用 Agent：

```
@Solar            → 完整流程
@Researcher       → 技术调研
@Architect        → 架构设计
@PM               → 产品验收
@Reporter         → 技术报告 (详细项目报告)
@Coder            → 代码实现
@Tester           → 测试
@Reviewer         → 代码审查
@Docs             → 文档
@Ops              → 构建部署
@Guard            → 规范检查
@Secretary        → 记录整理
@BenchmarkReporter → 测试报告 (结构化性能报告)
```

**识别规则:** 消息以 `@Agent` 开头时，直接调用该 Agent，跳过流程调度。

**响应格式:**
```
[@Agent名称]
─────────────
{响应内容}
```

## 一、流程阶段

```
P0 分析 → P1 研究 → P2 设计 → P3 实现 → P4 验证 → P5 收尾
         (可选)
```

| 阶段 | Agent | 触发条件 |
|------|-------|----------|
| P1 研究 | Researcher | 新技术/不确定方案 |
| P2 设计 | Architect + Guard | 中等/复杂任务 |
| P3 实现 | Coder + Guard | 需要写代码 |
| P4 验证 | Tester // Reviewer // Docs | 代码完成 |
| P5 收尾 | Ops → PM → Secretary | 验证通过 |

`//` = 并行 | `→` = 串行

## 二、复杂度路由

| 复杂度 | 判断标准 | 执行流程 |
|--------|----------|----------|
| 简单 | <50行, 单文件 | 直接做 |
| 中等 | 50-500行, 2-5文件 | P2→P3→P4 |
| 复杂 | >500行, 跨模块 | P1→P2→P3→P4→P5 |

## 三、并行规则

```
可并行: [Architect, Guard] | [Tester, Reviewer, Docs]
必须串行: Researcher→Architect→Coder→Tester
```

## 四、Gate 检查

| Gate | 位置 | 失败处理 |
|------|------|----------|
| G1 | P2后 | 重新设计 |
| G2 | P4后 | 返回P3修改 |
| G3 | P5后 | 迭代改进 |

**最大重试:** P2=2次, P3=3次, P4=2次

## 五、⚠️ 性能回归检查 (必须)

**每次代码变更必须检查:**

| 检查项 | 阻断条件 |
|--------|----------|
| 性能回退 | >5% 回退则阻止 |
| 算子丢失 | 优化算子未集成则阻止 |
| 优化缺失 | SIMD/并行被移除则阻止 |

**Guard 检查:**
- 最新算子文件是否被引用
- CMakeLists.txt 是否包含最新版本
- SIMD/Neon 优化代码是否存在

**Tester 检查:**
- 运行 benchmark 对比历史版本
- 维护 `.solar/performance.md` 记录
- 性能下降则阻止提交

## 五、资源限制

| 复杂度 | Token | Agent | 重试 |
|--------|-------|-------|------|
| 简单 | 5K | 0 | 1 |
| 中等 | 30K | 4 | 2 |
| 复杂 | 100K | 10 | 3 |

**超限停止，汇报进展。**

## 九、会话恢复

### 保存检查点 `/save`
以下时机主动保存:
- 完成一个阶段 (P1-P5)
- 做出重要决策
- 任务暂停或切换

### 恢复会话 `/restore`
- **只读** `.solar/session.md`
- **禁止** 重新读取所有源文件
- **禁止** 重新分析项目结构
- 确认后才按需读取必要文件

## 十、禁止行为

- 让用户手动调 Agent
- 超限继续执行
- 跳过 Gate 检查
- 重复读同一文件
- 恢复时读取大量文件
- **硬编码** (魔数/路径/URL/阈值)

## 十一、代码规范

**禁止硬编码，必须:**
```
魔数      → 定义常量 (const/constexpr/#define)
路径      → 使用配置或环境变量
URL/端口  → 可配置
阈值/参数 → 提取为常量或配置项
```

|------|------|------|
