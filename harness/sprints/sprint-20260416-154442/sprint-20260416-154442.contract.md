### 目标架构

```
┌─────────────── Session Layer ──────────────┐
│  events.jsonl (append-only)                │
│    {ts, event, sid, data...}               │
│    {ts, event, sid, data...}               │
│                                            │
│  session.sh API:                           │
│    append <sid> <json>                     │
│    get <sid> --from=N --to=M               │
│    tail <sid> N                            │
└────────────────────────────────────────────┘
              ↕
┌─────────────── Harness Layer ──────────────┐
│  coordinator.sh  (脑, 无状态)              │
│    从 events.jsonl 读取位置                 │
│    推导下一步动作                           │
│    派发 dispatch.md → pane                  │
│                                            │
│  solar-harness wake <sid>  (恢复入口)       │
│    读 events.jsonl 尾部                     │
│    重建 tmux pane (如果死了)                │
│    派发续接指令                             │
│                                            │
│  watchdog.sh  (守护)                        │
│    每 30s 检查 coordinator PID              │
│    死了自动 wake                            │
└────────────────────────────────────────────┘
              ↕
┌─────────────── Sandbox Layer (规范) ───────┐
│  (本 Sprint 只定义接口, 不实现)             │
│                                            │
│  sandbox-tool.md 规范:                      │
│    sandbox.execute(cmd, args)              │
│    sandbox.read_file(path)                 │
│    sandbox.write_file(path, content)       │
│                                            │
│  未来: Claude 通过 MCP 调用, 不直接 Bash    │
└────────────────────────────────────────────┘
```

### wake 算法

```
wake(sid):
  events = session.get(sid, --tail=50)
  last = events[-1]

  # 从最后事件推导下一步
  match last.event:
    "contract_created"  → 重派给规划者
    "done_filled"       → 重派给建设者 (planning)
    "plan_submitted"    → 重派给审判官 (审批计划)
    "plan_approved"     → 重派给建设者 (实现)
    "reviewing"         → 重派给审判官 (评审)
    "failed_review"     → 重派给建设者 (修复)
    "passed"            → noop (已完成)

  # 确保 tmux 存在
  if not tmux.has_session: tmux.rebuild_from_layout()

  # 确保 coordinator 运行
  if not coordinator.alive: coordinator.restart()

  # 幂等检查: 是否已经 wake 过且无新事件
  if last.event == "waked" and no_new_activity: return

  # 重新派发
  dispatch_to_pane(推导出的目标, 指令)

  # 记录 wake 事件
  session.append(sid, {"event":"waked", "by":"watchdog"})
```

### 死代码审计方法

```
对每个 handler 做历史统计:

handler_name       | 调用次数 | 最近调用    | 结论
handle_active      | 10      | 2026-04-15 | 活跃
handle_planning    | ?       | ?          | 待查
handle_approved    | ?       | ?          | 待查
handle_reviewing   | ?       | ?          | 活跃
handle_failed_review | ?     | ?          | 待查
handle_passed      | 10      | 2026-04-15 | 活跃

规则:
  从未触发      → legacy 候选 (标记, 不删)
  3 次以下      → 低频 (保留+警告)
  > 5 次        → 活跃
```
