## 约束

> 规划者填写

### 意图识别
- 触发词匹配必须**精准不误判**: "今天搬家了" 不得触发 migrate (只匹配明确词如"迁移到"/"打包 Solar"/"从 bundle"等)
- 参数提取后必须在 stdout 回显给监护人二次确认
- 主脑收到 migrate 意图时, **destructive 操作 (import/rollback/deploy) 必须监护人输入"确认"/"y"/"干"才执行**; safe 操作 (export/verify) 可直接执行

### 安全
- ssh host 参数必须做基本校验: `[a-zA-Z0-9_@.:-]+` 白名单, 防止命令注入
- bundle 路径必须存在 (import 前 `[[ -f "$bundle" ]]` 检查) 或是合法 `ssh://` URL
- deploy 预检 `ssh <host> 'echo ok'` 超时 10s, 失败不继续

### 兼容
- 不破坏现有 9 个 UserPromptSubmit hooks 行为 (新 hook 段独立, 在现有 Phase 间插入)
- 不破坏 sprint-20260422-192238 / 162434 / 151839 等已 passed 功能
- solar-harness CLI 本身不改 (沿用 sprint-162434/151839 的子命令)
