### PLANNER-INBOX 通知问题分析

合约说"漏报"，实际不是漏报，而是：
1. `notify_planner_inbox` L1210 写的格式是 `"Sprint PASSED: ${title}"`，不含 sprint ID
2. 验证命令 `grep sprint-20260501-213735 PLANNER-INBOX.md` 匹配不到（行里只有 title 没有 sid）
3. **修复**: `notify_planner_inbox` 的 msg 中加入 `${sid}`

但等一下，`handle_passed` L1382 调用时只传了 title：
```bash
notify_planner_inbox "Sprint PASSED: ${title}"
```
需要把 sid 也传进去。

同时 `handle_failed_review` (L1195) 也只传了 sid 不传 title，已经包含 sid：
```bash
notify_planner_inbox "Sprint FAIL: ${sid} Round ${round}"
```
FAIL 行有 sid，PASSED 行没有。

### 修复方案

在 `handle_passed` 的 L1382 改为：
```bash
notify_planner_inbox "Sprint PASSED: ${sid}: ${title}"
```

这样 PLANNER-INBOX 行中同时包含 sid 和 title，grep sid 可匹配。
