## D6 详解: CLAUDECODE Guard 环境冲突

**现象**:
```bash
$ bash runner.sh --dry-run --sprint-id sprint-test-eval
Guard FAIL: polluted env var: CLAUDECODE
```

**根因**: runner.sh Guard 2 检测 `CLAUDECODE|CLAUDE_CODE_*` 环境变量，若存在则 exit 1。当前 Claude Code 环境有:
- `CLAUDECODE`
- `CLAUDE_CODE_ENTRYPOINT`
- `CLAUDE_CODE_EXECPATH`

**设计意图**: 防止在 Claude Code TUI 内启动 Codex app-server，这是正确的安全考虑。

**问题**: P0 dry-run 不启动 Codex，此 guard 对 dry-run 模式过于严格。dry-run 只写文件，不需要这个 guard。

**建议修复**: Guard 2 仅在 `DRY_RUN=0` (即 `--unsafe-run-codex`) 时生效。dry-run 不需要环境检测。

**影响评估**: 低。测试脚本单独验证了 proof artifacts 写入正确性（绕过了 runner.sh 的 guard）。生产使用时 dry-run 会在 Claude Code 环境内失败。
