## 约束

> 规划者填写

约束 (P0/P1):
1. 必须真实接电: 不允许 mock 实现 (no-mock 铁律), solar-cache/solar-verify/phase-state-machine 必须能在新 sprint 上端到端跑通
2. 向前兼容: 旧 sprint (drafting/active/passed) 不受影响, status.json 缺 phase 字段时默认 phase=legacy
3. SDD-CACHE 严守'origin-validated': 缓存命中前必须 revalidate (HTTP If-None-Match 或 SQLite mtime+sha256), 禁止时间窗口缓存 (TTL)
4. Verification gates 必须可机械执行: cmd 不准包含 'TODO/手动检查/参考', expected_exit 必须是数字, output_pattern 必须是 grep -E 兼容 regex
5. bash 兼容性: 必须能在 bash 3.2 (macOS 默认) 和 bash 5.x 都跑 (避免 declare -A)
6. 性能: cache 命中时 revalidation HEAD 请求 < 500ms; phase transition < 100ms
7. 安全: cache 目录权限 700, bundle 中不含密钥/cookie/token
8. 文档双签: 改 DISPATCH-PROTOCOL.md 必须同时改 MEMORY.md 一条记录
9. 不写代码 (规划者), 全交建设者实现; Solar 自身代码不委派老专家 (老专家看不到本地文件)
