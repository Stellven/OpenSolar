## 额外发现

### 1. DISPATCH_MOCK 是既有测试 shortcut (低风险)
coordinator.sh:542-543 的 `DISPATCH_MOCK` 是 sprint-104819 引入的测试用环境变量 shortcut, 非 mock 实现. test-dispatch-overlap.sh 依赖它. 不阻塞 D7.

### 2. D1 缺少 "改 >= 3 个不同模块" 规则 (合约偏离, 低风险)
合约 D1 (b) 说 "Done >= 7 或改 >= 3 个不同模块: mixture", 但 select_topology 只检查 done_count ≥ 7, 不检查模块数. 这与 plan 审批时一致 — 模块数在 plan 阶段无法可靠提取. 实际效果: 只有大 Done 数的 sprint 才触发 mixture, 合理的保守策略.

### 3. merge_handoffs 用 awk+tolower (macOS 兼容)
handoff 提到 "sed !p 语法在 macOS 不兼容", 已改用 awk+tolower. 正确的 macOS 兼容处理.

### 4. building_parallel 超时机制缺失 (低风险)
handle_building_parallel 只检查 handoff 文件是否存在, 无超时. 如果一个 builder 卡住, 另一个的 handoff 会一直等. 建议未来 sprint 加超时保护, 不阻塞本次.
