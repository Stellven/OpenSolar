### D1: Debug-only prompt capture mechanism

**入口**: `src/omlx/scheduler.py` 的 `add_request()` 方法 (line ~3396)

在 `add_request()` 的 tokenization 完成后 (line ~3415) 加一个轻量门控调用:

```python

try:
    from .diagnostics.prompt_capture import capture_if_enabled
    capture_if_enabled(request)
except ImportError:
    pass
```

**`prompt_capture.py` 核心逻辑**:

```python
import os
import json
import hashlib
import time
from pathlib import Path
from typing import Optional, List

_capture_dir = Path.home() / ".thunderomlx" / "debug" / "prompt-captures"

def _is_enabled() -> bool:
    val = os.environ.get("THUNDEROMLX_DEBUG_PROMPT_CAPTURE", "0")
    return val in ("1", "true", "yes")

def capture_if_enabled(request) -> None:
    """Gate: returns immediately if env var not set. Zero overhead path."""
    if not _is_enabled():
        return
    _do_capture(request)

def _do_capture(request) -> None:
    """Full capture: write JSONL + artifact files."""
    capture_id = f"{request.request_id}_{int(time.time()*1000)}"
    ts = time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime())

    prompt_text = request.prompt if isinstance(request.prompt, str) else ""
    prompt_tokens = request.prompt_token_ids or []
    prompt_chars = len(prompt_text)

    # Compute message boundaries
    message_boundaries = []
    if hasattr(request, 'messages') and request.messages:
        offset = 0
        for msg in request.messages:
            content = msg.get("content", "")
            if isinstance(content, str):
                # Approximate boundary by token count
                msg_tokens = request.prompt_token_ids[offset:offset+len(request.tokenizer.encode(content))] if hasattr(request, 'tokenizer') and request.tokenizer else []
                message_boundaries.append(offset + len(msg_tokens))
                offset += len(msg_tokens)

    # System hash
    system_hash = hashlib.sha256(
        (request.messages[0].get("content","") if request.messages and request.messages[0].get("role")=="system" else "").encode()
    ).hexdigest()[:16]

    # Artifact paths
    _capture_dir.mkdir(parents=True, exist_ok=True)
    token_ids_path = _capture_dir / f"{capture_id}_token_ids.npy"
    raw_prompt_path = _capture_dir / f"{capture_id}_raw.txt"
    block_hashes_path = _capture_dir / f"{capture_id}_block_hashes.json"

    # Write artifacts
    import numpy as np
    np.save(str(token_ids_path), np.array(prompt_tokens, dtype=np.int32))
    raw_prompt_path.write_text(prompt_text)

    # Compute block hashes (same algorithm as paged_cache.py)
    block_size = 256  # matching the observed 17408 / 68 = 256
    block_hashes = []
    for i in range(0, len(prompt_tokens), block_size):
        block_tokens = prompt_tokens[i:i+block_size]
        parent = block_hashes[-1] if block_hashes else None
        h = hashlib.sha256(
            (parent or "") + ",".join(str(t) for t in block_tokens).encode()
        ).hexdigest()
        block_hashes.append(h)
    block_hashes_path.write_text(json.dumps(block_hashes))

    # Write JSONL record
    record = {
        "capture_id": capture_id,
        "timestamp": ts,
        "prompt_chars": prompt_chars,
        "prompt_tokens": len(prompt_tokens),
        "message_boundaries": message_boundaries,
        "system_hash": system_hash,
        "token_ids_path": str(token_ids_path),
        "raw_prompt_path": str(raw_prompt_path),
        "block_hashes_path": str(block_hashes_path),
    }
    jsonl_path = _capture_dir / f"{capture_id}.jsonl"
    jsonl_path.write_text(json.dumps(record) + "\n")
```

**零开销保证**: `_is_enabled()` 检查 `os.environ.get()` — CPython 中这是 O(1) dict lookup，<100ns。当 disabled 时，整个 `capture_if_enabled` 从 import 到 return 不超过 1us，对于 91K-token prefill (数十秒级) 完全可忽略 (<0.001%)。

### D2: JSONL record with artifact files

每个 scheduled request 写一个 JSONL 文件 `~/.thunderomlx/debug/prompt-captures/<capture_id>.jsonl`，包含 8 个必填字段 + 3 个 artifact 文件实际存在于磁盘。

capture_id 格式: `{request_id}_{timestamp_ms}`

### D3: Token-level diff utility

**脚本**: `scripts/diff_prompt_captures.py`

输入: 两个 JSONL capture 文件路径
输出: 结构化 diff 报告 (stdout) + 诊断报告文件

```
python scripts/diff_prompt_captures.py <capture_a.jsonl> <capture_b.jsonl>
```

核心算法:
1. 加载两个 capture 的 token_ids (.npy) 和 block_hashes (.json)
2. 逐 token 比较，找 first_mismatch_token_index
3. 从 token_index 计算 block_index = token_index // 256
4. 找 nearest_message_boundary
5. 提取 left/right context (各 ≥10 tokens)
6. 计算 prefix_match_percentage = first_mismatch / total * 100
7. 调用 cache_probe (D4) 检查相关 blocks
8. 生成诊断报告 (D5)

### D4: Cache lookup probe (read-only)

**模块**: `src/omlx/diagnostics/cache_probe.py`

对指定 block_hashes 列表，逐个查询 PagedCacheManager 的 `cached_block_hash_to_block` (只读，不调用 allocate/evict):

```python
def probe_blocks(
    block_hashes: List[str],
    cache_manager,  # PagedCacheManager
    ssd_cache=None,  # Optional PagedSSDCacheManager
) -> List[dict]:
    results = []
    for bh in block_hashes:
        entry = {
            "block_hash": bh,
            "metadata_present_in_paged_cache": False,
            "payload_present_on_ssd": False,
            "cache_tier": "missing",
            "eviction_status": "unknown",
        }
        # Read-only: only check existence, never write
        block = cache_manager.cached_block_hash_to_block.get_block(
            bytes.fromhex(bh)
        )
        if block is not None:
            entry["metadata_present_in_paged_cache"] = True
            entry["cache_tier"] = "l1"
            entry["eviction_status"] = "active" if block.ref_count > 0 else "evictable"
        # SSD check (if available)
        if ssd_cache is not None:
            try:
                ssd_path = ssd_cache._get_block_path(bh)
                if ssd_path and Path(ssd_path).exists():
                    entry["payload_present_on_ssd"] = True
                    entry["ssd_path"] = str(ssd_path)
                    if entry["cache_tier"] == "missing":
                        entry["cache_tier"] = "ssd"
            except Exception:
                pass
        results.append(entry)
    return results
```

**只读保证**: 只调用 `get_block()` 和文件系统检查，不调用任何写操作。

### D5: Diagnosis report

diff utility 输出 `diagnostics/cache-miss-report-<timestamp>.md`，包含:

1. **Verdict**: H1/H2/H3/H4/inconclusive
   - H1 (dynamic injection): message_boundaries 内有变化 → system/user message 内容不同
   - H2 (eviction/persistence gap): hash 匹配但 metadata 不在 paged_cache
   - H3 (cache write/read bug): hash 相同但 block 内容不同
   - H4 (genuine new content): token 内容确实不同 = 物理限制

2. **Evidence table**: 每个 hypothesis → supporting data / refuting data

3. **Raw text snippet**: first mismatch ±200 chars, 高亮 mismatch 位置

4. **Cache probe results**: 6 blocks 的详细状态

5. **Recommendation**: next action 建议

## 风险点

1. **block_size 不一致**: paged_cache.py 默认 `block_size=64`，但合约提到 block 67 = token 17408 → 17408/67 ≈ 259.8，接近 256。需要运行时读取 `cache_manager.block_size` 而非硬编码。

2. **request.prompt 可能为 list[int]**: `add_request()` line 3414 显示 prompt 可以是 `list[int]` (非 string)。`_do_capture()` 需要处理两种类型。

3. **messages 可能为 None**: `request.messages` 可能为空列表或 None。需要 null-safe 处理。

4. **numpy 依赖**: 项目已有 numpy 使用 (server.py, two_tier_cache_wrapper.py)，但需要确保 tests 也能 import numpy。

5. **SSD cache 可选**: `PagedSSDCacheManager` 是 try-import，probe 需要优雅处理 SSD cache 不可用的情况。

6. **threading safety**: `add_request()` 可能在多线程环境调用。capture 写文件用独立路径 (capture_id 唯一)，无竞态。

7. **大 prompt 的内存开销**: 91K token 的 `.npy` 文件 ≈ 360KB，`.txt` 文件可能数 MB，block_hashes ≈ 数 KB。总计每次 capture < 10MB，可接受。
