### Part A: 差分备份 (5 条)

- [ ] **D1 差分扫描**: 新增 `migrate/diff-scan.sh` 或 import.sh 内 `scan_diff_before_import()`: import 展开前扫描目标机 `~/.solar` + `~/.claude` 与 bundle 清单对比, 分类: (a) added - bundle 有目标没有 (b) modified - 两边都有但 SHA256 不同 (c) untouched - 相同不备份。**性能**: 10k 文件 < 60 秒 (Python concurrent.futures 或 parallel md5)。跳过 `.git/` `node_modules/` `__pycache__/` `*.pyc` `.DS_Store`

- [ ] **D2 差分备份物料**: 展开前对 modified 类文件, 目标机原文件 `cp -p` 到 `~/solar-diff-backup-<ISO_TS>/originals/<相对路径>/` 保留权限; added 类不备份 (目标本无); diff-manifest.json 记录每 modified 的 target_sha256_before 便于恢复校验

- [ ] **D3 `rollback --diff` 恢复**: `solar-harness migrate rollback --diff [--backup-dir <path>]`: (a) 找最近 `~/solar-diff-backup-*/` (按 ts); (b) 读 manifest; (c) modified → 从 originals/ 恢复 + 校验 SHA256 == before; (d) added → 从目标路径删除; (e) 打印摘要 N 还原 + M 删除 + 0 失败

- [ ] **D4 rollback 双模式兼容**: `rollback --full` 走原全量备份 (sprint-162434 已做); `rollback --diff` 走本 Sprint 新增; 无参数默认先 --diff, fallback --full; rollback.sh 头部加子命令路由。不破坏 sprint-162434 的 D9 兼容性

- [ ] **D5 import 默认双备份**: `migrate import <bundle>` 默认生成 full + diff; `--skip-diff-backup` 或 `--skip-full-backup` 可选择; 日志打印两个备份路径 + 大小

### Part B: SSH 远程模式 (5 条)

- [ ] **D6 `export --push <user@host:path>`**: export 打包完后直接 `scp` (或 `rsync --partial --append`) 推送到远程, 成功后可 `--cleanup-local` 删本机 bundle 节省空间; secrets 包同步推送 (加密态); 支持 ssh key 认证, 不支持密码; bundle 路径尾部没 / 时自动补

- [ ] **D7 `import ssh://<user@host>/<path>`**: import 识别 `ssh://` 前缀时, 先 `scp` 或 `rsync` 把远程 bundle 拉到本地 `/tmp/solar-import-<ts>.tar`, SHA256 校验通过后走原 import 流程; 支持 `--keep-local` 保留本地副本

- [ ] **D8 `migrate deploy <user@host>` 一键部署**: 整合流程: (1) 本机 export --push `<host>:~/solar-bundles/`; (2) `ssh <host> "bash -lc 'solar-harness migrate import ~/solar-bundles/latest.tar'"`; (3) `ssh <host> "solar-harness doctor"`, doctor 非零 → 远程自动 rollback --diff + 本机 loud fail; 全程 stdout 实时转发, 用户能看到远程机器的进度; 默认不覆盖目标机已有 ~/.solar (除非 --force)

- [ ] **D9 `rollback --remote <user@host>`**: 本机触发远程 rollback, 走 `ssh` 执行 `solar-harness migrate rollback --diff` (默认 diff 模式, `--full` 切换); 远程输出实时回显; 失败 (remote exit != 0) 本机非零退出

- [ ] **D10 SSH bootstrap 脚本**: 新增 `migrate/ssh-bootstrap.sh <user@host>`: 目标机没装 solar-harness 时 (首次迁移), 本机 `scp -r ~/.solar <host>:~/` + `ssh <host> 'bash ~/.solar/bin/install-mac.sh'` (或等价命令) 把远程种子; 种子完成后 `ssh <host> 'solar-harness --version'` 验证

## 约束

> 规划者填写

### 差分备份
- 差分目录命名 ~/solar-diff-backup-<ISO_TS>/ (与 full backup 目录区分)
- diff-manifest.json 必须合法 JSON (jq . 通过), 含 version/created_at/stats/added/modified 字段
- SHA256 用 Python hashlib (性能更好) 而非 shasum (启动慢)
- 扫描跳过: .git/ node_modules/ __pycache__/ *.pyc *.pyo .DS_Store *.swp
- 性能: 10k 文件 < 60s, 1k 文件 < 6s (冒烟断言)
- 并行度: Python concurrent.futures, 默认 4 进程, --jobs N 可覆盖
- 差分与全量并存, 不互相影响

### SSH 远程
- 只支持 ssh key 认证, 不支持密码交互
- 所有 ssh 调用加 `-o BatchMode=yes -o StrictHostKeyChecking=accept-new` 避免交互 prompt
- 大文件传输优先 rsync --partial --append --progress (断线续传), 无 rsync fallback scp
- 远程命令执行用 `ssh <host> "bash -lc '<cmd>'"` (login shell, 保证 PATH 有 solar-harness)
- stdout/stderr 实时转发 (不缓冲整段)
- deploy 失败自动触发远程 rollback --diff, 本机 loud fail
- secrets 包传输前必须已加密 (export --include-secrets 产出的 .enc), 不得明文过 SSH
- ssh_config 里的 host 别名同样支持 (不强制 user@host, 允许纯 <alias>)

### 通用
- 所有 shell 脚本 bash -n + shellcheck (如装) 通过
- 不破坏已 passed 的 sprints: 172812/192238/203859/211820/222017/062851/162434
- **审判官必须 smoke test + 否证尝试** (sprint-062851 D4/D5 铁律)
- round 上限 5 (5 轮仍未过 loud fail, 不无限循环)
- 所有新增 shell 加 shebang `#!/usr/bin/env bash`
- 兼容 macOS 15+ / Apple Silicon & Intel (grep/sed/find/xargs BSD 版本语法)
