## Definition of Done

- [ ] **D1 git-filter-repo 安装 + 验证**
  - `brew install git-filter-repo`
  - `which git-filter-repo` 返回非空
  - `git filter-repo --version` 输出版本号

- [ ] **D2 三仓库 P0 文件整体删除(filter-repo --invert-paths --path)**
  - 必删模式: `.env` `.env.*`(不删 `.env.template` `.env.example`) `client_secret_*.json` `*credentials*.json` `*-private.*`
  - 三仓库分别处理: ~/.claude, ~/Solar, ~/Solar-MAX
  - 每仓库执行前先 cd 进去, `git filter-repo --invert-paths --path .env --path-glob 'client_secret_*.json' --path-glob '*credentials*.json' --force`
  - 验证: 每仓库 `git log --all --oneline -- .env` 应该返空; `git log --all --oneline -- '*client_secret*'` 也返空

- [ ] **D3 硬编码 API key 内容替换(filter-repo --replace-text)**
  - 准备替换规则文件 `/tmp/filter-repo-replace.txt`, 每行 `regex==>REDACTED` 格式:
    - `regex:sk-ant-[A-Za-z0-9_-]{20,}==><REDACTED:ANTHROPIC_KEY>`
    - `regex:sk-or-[A-Za-z0-9]{40,}==><REDACTED:OPENAI_KEY>`
    - `regex:AKIA[A-Z0-9]{16}==><REDACTED:AWS_KEY>`
    - `regex:ZHIPU_API_KEY=["\']?[a-zA-Z0-9._-]+["\']?==>ZHIPU_API_KEY="<REDACTED>"`
    - `regex:DEEPSEEK_API_KEY=["\']?sk-[a-zA-Z0-9]+["\']?==>DEEPSEEK_API_KEY="<REDACTED>"`
    - `regex:GOOGLE_API_KEY=["\']?[a-zA-Z0-9_-]+["\']?==>GOOGLE_API_KEY="<REDACTED>"`
  - 三仓库 `git filter-repo --replace-text /tmp/filter-repo-replace.txt --force`
  - 验证: 每仓库 `git log -p --all | grep -E "sk-(ant|or)-[A-Za-z0-9]{20,}"` 返空

- [ ] **D4 force push 三仓库到 GitHub origin main**
  - 注意: filter-repo 会把 origin remote 删除 (这是它的安全机制), 必须先重加: `git remote add origin <url>`
  - 三仓库 url:
    - ~/.claude: https://github.com/lisihao/solar.git (remote 名 'solar' 不是 'origin')
    - ~/Solar: git@github.com:lisihao/Solar.git
    - ~/Solar-MAX: git@github.com:lisihao/Solar.git
  - `git push --force-with-lease <remote> main` (~/.claude 的 remote 是 'solar')
  - 验证: 三仓库 `git ls-remote <remote> main` 返回的 hash == 本地 `git rev-parse HEAD`

- [ ] **D5 GitHub 验证 + GITHUB-PURGE-NOTICE.md**
  - 用 GitHub API 查 main commit 历史: `curl -s 'https://api.github.com/repos/lisihao/solar/commits?per_page=1' | jq '.[0].sha'` 跟本地一致
  - GitHub 上 `git log -p main -- .env` 应该返空 (用 git clone --depth=full https://github.com/lisihao/solar.git /tmp/verify-clone && cd /tmp/verify-clone && git log --all -- .env 验证)
  - 写 ~/Solar/GITHUB-PURGE-NOTICE.md (≥ 30 行): 解释发生了什么 / fork 用户怎么重新 clone / 本仓库历史哪一天被清理 / 联系方式
  - 提交 NOTICE 到 main 并 push (这是新 commit, 不需要 force)
  - 验证: GitHub 上 NOTICE 文件可访问, 内容包含"git purge" "rebase" "fork"

## Constraints

1. **本机文件神圣不可侵犯**: filter-repo 只动 .git 历史, 绝对不能 rm working tree 的 .env / .zshrc / 任何当前文件 (filter-repo 默认不会 rm working tree, 但跑完后要验证 `ls ~/.zshrc ~/.solar/brain-router/.env` 仍存在)
2. **真实接电 (no-mock)**: 所有 git 操作真跑, 输出必须粘 stdout 给审判官 grep
3. **mirror backup 不可删**: ~/.solar/backups/*-pre-purge-* 是恢复保险, 整个 sprint 期间不动
4. **失败响亮**: filter-repo 任一步失败, 立刻 stop, 不能半途;尤其 D4 push 失败必须 stderr + 桌面通知
5. **force push 顺序**: D2 → D3 → D4 严格顺序, 不能并行 (filter-repo 是 destructive)
6. **重加 remote**: filter-repo 删除 origin 是 feature, 必须先 `git remote add` 才能 push
7. **D4 push 一次只一个仓库**: ~/.claude / ~/Solar / ~/Solar-MAX 串行 push, 一个 fail 立刻停
8. **bash 5.x + jq + git filter-repo**
9. **不写代码 (规划者)**: 全交建设者 Sonnet 4.5 实现
10. **D5 必须粘 stdout**: 审判官 grep `git ls-remote` 输出 + GitHub API 查询输出
