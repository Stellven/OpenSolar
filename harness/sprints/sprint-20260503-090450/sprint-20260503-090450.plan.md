### D1: Topology 路由 — handle_active 入口

**新工具函数** (放在 `get_field` 附近, ~line 95):

```bash
get_topology() {
  # 从合约 frontmatter 读 topology: 字段, 默认 standard
  local sid="$1"
  local cf="$SPRINTS_DIR/${sid}.contract.md"
  [[ -f "$cf" ]] || { echo "standard"; return; }
  # 只在头部 frontmatter 区 (前 30 行) 找 topology:
  local val
  val=$(awk '/^---$/{c++; next} c==1' "$cf" 2>/dev/null \
        | grep -E '^topology:' | head -1 \
        | sed 's/^topology:[[:space:]]*//' | tr -d '"' | tr -d "'")
  # 校验白名单, 非法值降级 standard
  case "$val" in
    standard|deliberation|research) echo "$val" ;;
    *) echo "standard" ;;
  esac
}
```

### D2: handle_architect_reviewing + 模板

```bash
handle_architect_reviewing() {
  local sid="$1" sf="$2"
  log "${C}deliberation: 派给 architect 二审${N}"

  generate_architect_dispatch "$sid" "deliberation" "二审 evaluator 通过的实现"
  dispatch_to_pane "$PANE_ARCHITECT" "" "$sid"
  emit_event "$sid" "dispatched" "coordinator" \
    "{\"to\":\"architect\",\"topology\":\"deliberation\",\"task\":\"second_review\"}"
}

generate_architect_dispatch() {
  local sid="$1" topology="$2" task="$3"
  local tpl="$HARNESS_DIR/templates/architect-dispatch.md"
  local out="$SPRINTS_DIR/${sid}.dispatch.md"
  # 模板变量替换
  sed -e "s|{{SID}}|${sid}|g" \
      -e "s|{{TOPOLOGY}}|${topology}|g" \
      -e "s|{{TASK}}|${task}|g" \
      "$tpl" > "$out"
}
```

## 风险点

| 风险 | 严重度 | 缓解 |
|------|--------|------|
| frontmatter 解析: awk `'/^---$/{c++}'` 在 contract 头部第 1 行不是 `---` 时漏读 | 中 | `get_topology` 默认 standard, 解析失败不阻塞 |
| pane 3 同时被 architect 和 (旧的) builder2 占 | 低 | choose_builder_pane 已硬编码 pane 1, 不会冲突. 但要在文档强调 pane 3 不能再被任何 builder 路径用 |
| handle_passed 双跑: deliberation 第一次 set architect_reviewing, 第二次 (architect PASS 后) 真 finalize | **高** | 用 `architect_verdict` 字段做幂等 guard, 已在方案中处理 |
| save_state 写 architect_reviewing 到 cache, 主循环下轮以为状态变更 → 重派 architect | 中 | check_state_changed 会比对 cache, architect 改 status=passed 后会再触发 handle_passed (含 architect_verdict 字段, 跳过 deliberation 分支, 真 finalize) |
| watchdog 不知道 architect 在干活, pane 看起来 idle 长时间 → 误判死亡 respawn | 中 | architect 任务长 (opus 长链), 但 `is_claude_alive` 走子进程树检测, 不只看 idle. 已经稳过 builder 长任务 |
| research 拓扑跳过 evaluator → 没人 catch architect 错误 | 中 | research 用于"长链调研", 输出本来就是研究报告, 不是代码. 监护人最终把关 |
| frontmatter 字段冲突: 现有合约用 `name:` 不是 `title:` | 低 | get_topology 不依赖 title/name, 只看 topology |
