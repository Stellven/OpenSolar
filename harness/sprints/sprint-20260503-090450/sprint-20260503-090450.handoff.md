## 关键设计决策

### D1: deliberation 幂等 guard
handle_passed 用 `architect_verdict` 字段判断是否已过 architect:
- 第一次进 handle_passed: 无 architect_verdict → 改 status=architect_reviewing → return 0
- 第二次进 handle_passed (architect PASS 后): 有 architect_verdict → 跳过分支 → 真 finalize
防止 architect 二审后回到 handle_passed 时再次触发 deliberation 分支.

### D1: research 拓扑跳过 evaluator
research 用于长链调研, 输出是研究报告不是代码, 不需要 evaluator 质量门.
architect 写完 architect-review.md 直接改 status=passed.

### D3: pane 3 复用
PANE_ARCHITECT 与 PANE_BUILDER2 指向同一物理 pane (0.3).
choose_builder_pane 已硬编码返回 PANE_BUILDER (pane 1), 不会冲突.
watchdog respawn 用 second-builder profile (opus), 与 architect 需求等价.
