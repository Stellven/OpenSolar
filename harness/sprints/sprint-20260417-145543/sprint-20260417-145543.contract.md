## 需求

协调器体验升级，解决两个用户痛点:

**痛点 1**: 每 3 秒 `sleep 3` + `python3 json.load` 轮询 → 重、频繁、浪费
**痛点 2**: `tmux send-keys "读取并执行指令文件 /很长路径/dispatch.md 中的所有步骤"` → 80 个字符出现在 Claude 输入框，丑且容易被 thinking-state 吞

改为: **stat mtime 事件检测 + `/next` 短触发词 + hook 自动注入**

## Done 定义

- [ ] D1: 轮询改 mtime 检测 — 重写 coordinator.sh 主循环: 不再每轮 `python3 json.load` 解析 JSON，改为 `stat -f %m` 检测 sprints 目录的修改时间戳。只有 mtime 变化时才 `python3` 解析具体内容。轮询间隔从 3 秒改为 10 秒。**验收**: `time` 测 100 轮循环，stat 方式 < 1 秒，原 python3 方式 > 10 秒

- [ ] D2: 长指令改 `/next` 短触发词 — 修改 `dispatch_to_pane()`: 不再 `tmux send-keys "读取并执行指令文件..."`, 改为只发 `/next` 两个字符 + Enter。**验收**: `tmux capture-pane` 建设者 pane，输入框只出现 `/next`，不出现文件路径

- [ ] D3: `/next` hook 自动注入 — 在 `~/.claude/hooks/` 新增 `harness-next-hook.sh` (UserPromptSubmit): 检测用户输入是否为 `/next`，如果是则 (a) 找最新的 `*.dispatch.md` (b) 读取内容 (c) 输出为 `<intent-detected>` 标签注入 Claude 上下文。**验收**: 在建设者 pane 手动输入 `/next`，Claude 自动读取并执行 dispatch.md 内容

- [ ] D4: hook 注册到 settings.json — 把 `harness-next-hook.sh` 注册到 `~/.claude/settings.json` 的 UserPromptSubmit hooks 中。匹配条件: `prompt` 包含 `/next`。**验收**: `cat ~/.claude/settings.json | grep harness-next` 有输出

- [ ] D5: fswatch 可选加速 — 如果 `fswatch` 可用 (`which fswatch`)，coordinator 用 `fswatch --one-event` 替代 `sleep 10 + stat` 做真正的零轮询事件驱动。不可用则 fallback 到 stat + 10 秒间隔。**验收**: `brew install fswatch` 后协调器日志显示 "fswatch 模式"

- [ ] D6: 端到端验证 — 创建测试 Sprint → 建设者 pane 只收到 `/next` → hook 自动读 dispatch → 建设者开始工作 → 全程无长路径出现在任何 pane。协调器日志显示 mtime 检测而非 json.load

## 设计要点

### D1: stat mtime 替代 python3

```bash
# 原: 每轮 python3 json.load (慢, ~100ms/轮)
st=$(python3 -c "import json; print(json.load(open('$f')).get('status',''))")

# 新: stat 检测目录 mtime (快, ~1ms/轮)
last_mtime=0
while true; do
  current_mtime=$(stat -f %m "$SPRINTS_DIR" 2>/dev/null || echo 0)
  if [[ "$current_mtime" != "$last_mtime" ]]; then
    last_mtime="$current_mtime"
    # 有变化才 python3 解析
    process_sprint_changes
  fi
  sleep 10
done
```

### D2: dispatch_to_pane 改短

```bash
dispatch_to_pane() {
  local pane="$1" message="$2" sid="${3:-dispatch}"

  # 规划者静默
  if [[ "$pane" == "$PANE_PLANNER" ]]; then
    notify_planner_silently "$sid"
    return 0
  fi

  # 写 dispatch.md (不变)
  local instruction_file="$SPRINTS_DIR/${sid}.dispatch.md"
  [[ -n "$message" ]] && echo "$message" > "$instruction_file"

  # 只发 /next (2 字符, 不是 80 字符)
  tmux send-keys -t "$pane" "/next" Enter 2>/dev/null
}
```

### D3: harness-next-hook.sh

```bash
#!/bin/bash
# Hook: UserPromptSubmit — 检测 /next, 注入 dispatch.md

PROMPT="$CLAUDE_USER_PROMPT"
[[ "$PROMPT" != "/next" ]] && exit 0

# 找最新 dispatch.md
DISPATCH=$(ls -t ~/.solar/harness/sprints/*.dispatch.md 2>/dev/null | head -1)
[[ -z "$DISPATCH" ]] && echo "无待处理任务" && exit 0

# 读取并注入
CONTENT=$(cat "$DISPATCH")
SID=$(basename "$DISPATCH" .dispatch.md)

cat << EOF
<intent-detected type="harness_dispatch" skill="harness" sprint="$SID">
$CONTENT
</intent-detected>
EOF
```

