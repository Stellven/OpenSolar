## 2. 数据模型变更

### 2.1 status.json 新增字段

```json
{
  "id": "sprint-xxx",
  "status": "drafting",
  "auto_held": true,                        // ← NEW
  "auto_held_reason": "2 consecutive dispatch failures within 24h (rc_1, rc_1)",  // ← NEW
  "auto_held_at": "2026-05-09T12:50:00Z",   // ← NEW
  "auto_held_failure_count": 2              // ← NEW
}
```

### 2.2 失败计数文件 (新增, 在 ~/.solar/harness/.drafting-failure-counts/)

```
~/.solar/harness/.drafting-failure-counts/<sid>.<stage>.count
  内容: <ts_unix>:<count>
  例: 1715258400:2
  规则: ts > 24h 前 → 重置为 0; 计数 ≥ 2 → 触发 auto_held
```

不放 status.json 内是为了让原子计数读写不污染 status (json 频繁覆写有竞态)。

### 2.3 brief 标记识别 (D1)

`brief_has_hold_marker(sid)` 用 grep, 不引入 LLM:

```
匹配模式 (grep -E 任一即视为 hold):
  - 待[^[:space:]]*手动 active
  - 待监护人手动 active
  - \[DRAFTING-HOLD-NO-AUTODISPATCH\]
  - hold drafting
  - 禁止再 dispatch
```

## 3. 控制流 (handle_drafting 改造前后)

### Before (现状, 有病)

```
status=drafting → handle_drafting
  ├─ no req_file → dispatch PM → fail → mark_retry → 写 inbox cooldown → 900s 后再来
  ├─ req_file && no plan → dispatch planner → fail → mark_retry → 写 inbox cooldown → 900s 后再来
  └─ plan exists → promote active
```
**结果**: 失败永不 hold, 噪音 ∞

### After (A 路, 治本)

```
status=drafting → handle_drafting
  ├─ NEW guard 1: brief_has_hold_marker(sid) → log "skip (brief hold)" → return 0
  ├─ NEW guard 2: status_auto_held(sid) → log "skip (auto_held)" → return 0
  ├─ no req_file → dispatch PM → fail → mark_retry++
  │     └─ count ≥ 2 → set auto_held=true + skip inbox cooldown write
  ├─ req_file && no plan → dispatch planner → fail → mark_retry++
  │     └─ count ≥ 2 → set auto_held=true + skip inbox cooldown write
  └─ plan exists → promote active
```
**结果**: 第 2 次失败立刻 hold, 后续 0 inbox 噪音, 仅监护人 `solar-harness sprint activate <sid>` 能解锁

## 4. 解锁路径 (D5)

```bash
solar-harness sprint activate sprint-xxx
```

实现 (≤30 行 in `~/.solar/bin/solar-harness`):

```bash
# pseudo-code, 建设者实现
sprint_activate() {
  local sid="$1"
  local sf="$HARNESS_DIR/sprints/${sid}.status.json"
  [[ -f "$sf" ]] || { echo "[err] sprint not found"; return 1; }

  python3 -c "
import json, datetime, sys
sf = '$sf'
d = json.load(open(sf))
was_held = d.get('auto_held', False)
d['auto_held'] = False
d.pop('auto_held_reason', None)
d['updated_at'] = datetime.datetime.now(datetime.UTC).strftime('%Y-%m-%dT%H:%M:%SZ')
d.setdefault('history', []).append({
    'ts': d['updated_at'],
    'event': 'manual_activate_unlock',
    'by': 'guardian',
    'was_auto_held': was_held
})
json.dump(d, open(sf, 'w'), indent=2)
print(f'[ok] {sid} unlocked (was_auto_held={was_held})')
"

  # 清失败计数
  rm -f "$HARNESS_DIR/.drafting-failure-counts/${sid}".*.count

  # touch 触发 coordinator 立即评估
  touch "$sf"
}
```
