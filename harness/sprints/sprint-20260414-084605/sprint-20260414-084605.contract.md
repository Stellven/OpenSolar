## 需求

实现 SkillRL Phase 2: 在 skill-retriever 的 `retrieveByLayer()` 中注入 Q-weighted 检索，并新增 `skill_feedback` MCP 工具用于记录技能使用反馈。

**目标**: 让技能检索结果不再只靠 BM25 静态分数，而是叠加历史表现的 Q-value (Thompson Sampling)，使高表现技能排名上升，低表现技能排名下降。

## Done 定义

- [ ] **D1 (功能-Q检索)**: `retrieveByLayer()` 在 RRF 融合阶段叠加 Q-weighted 分数，融合公式为 `final_score = rrf_score * (1 - q_weight) + q_score * q_weight`，其中 `q_weight` 可配置 (默认 0.3)
- [ ] **D2 (功能-Thompson)**: Q-score 来自 `skill_beliefs` 表的 `Beta(alpha, beta)` Thompson Sampling，采样使用 Box-Muller 或等效方法，sample_count=0 时退化为均匀先验 (alpha=1, beta=1)
- [ ] **D3 (功能-feedback工具)**: 新增 MCP 工具 `skill_feedback`，接受参数 `{skill_name: string, task_type: string, success: boolean, quality?: number}`，更新 `skill_task_performance` 和 `skill_beliefs` 两张表
- [ ] **D4 (兼容性)**: Q-weighted 检索对无历史数据的技能 (新技能/冷启动) 不产生负面影响 — 无数据时 Q-score = 0.5 (中性)，不改变原有 RRF 排序
- [ ] **D5 (性能)**: 检索延迟增加 < 50ms (SQLite 查询 + Thompson 采样)，通过 `console.time` 测量
- [ ] **D6 (验证)**: 至少 3 个端到端测试场景通过:
  - 场景 A: 有历史高分技能 → 排名应上升
  - 场景 B: 有历史低分技能 → 排名应下降
  - 场景 C: 无历史技能 → 排名不变 (与纯 BM25 一致)

**Thompson Sampling 伪代码**:
```python
function sample_beta(alpha: float, beta: float) -> float:
    # Box-Muller 或 jStat 等价
    # alpha=1, beta=1 时 → 均匀分布 [0,1]
    return Beta(alpha, beta).sample()

function get_q_score(skill_name: str, task_type: str) -> float:
    row = db.query("SELECT alpha, beta FROM skill_beliefs WHERE skill_name=? AND task_type=?", [skill_name, task_type])
    if not row: return 0.5  # 冷启动中性
    return sample_beta(row.alpha, row.beta)
```
