## 约束

### 安全
- Secrets 默认**不**打进主 bundle, 只在 `--include-secrets` 时单独加密分包
- AES-256-CBC (openssl enc) 或 GPG 对称加密, 密码**仅**通过 `--password <pw>` 或 stdin 传入, **不写入** 任何日志/meta/临时文件
- tar 打包完成后 `unset PASSWORD` + `export HISTCONTROL=ignorespace` 前缀调用, 避免命令历史泄露
- SSH 私钥 (`id_*`, `*.pem`, `*_rsa`) 必须在 secrets 分包内, 权限展开后强制 `chmod 0600`
- bundle-meta.json 标注 `has_secrets: true|false` + `secrets_encrypted: true|false`, 用于 import 时判断

### 跨机兼容
- **路径替换**: import 扫描 `/Users/<src_user>/` → `/Users/<dst_user>/` 仅在文本文件 (含扩展名 .md/.sh/.json/.py/.ts/.yml/.conf/.plist/.zshrc/.bashrc/无扩展的 rc 文件); 二进制文件 (含 .db/.tar/.gz/.png/.jpg/.pyc) 跳过
- **架构差异**: Intel ↔ Apple Silicon 时, brew bundle 目标机需先装 Rosetta 2 或用 arm64 版本; import 时 `arch` 检测不一致则 loud warn
- **macOS 版本差异**: 只做 Darwin 24+ (macOS 15+) 验证, 更老版本不保证 (bundle-meta.json 记录源机 `sw_vers`)
- **bash 版本**: 目标机必须先 `brew install bash` (>= 4.x), 否则 doctor 直接 fail

### 数据完整性
- 所有打包文件 SHA256 入 bundle-meta.json 的 `files_hash` 字典, import 时逐文件校验
- bundle 外层也有 SHA256 单独发布 (stdout 打印 + 伴生 `.sha256` 文件), 防传输损坏
- SQLite 数据库 (solar.db, MemPalace ChromaDB) 打包前执行 `sqlite3 .backup` 生成一致性快照, 不直接 cp 活文件
- ChromaDB 目录打包前需 pkill 相关进程或用 fsync 确保刷盘

### 幂等 & 回滚
- import 前必须备份目标机已存在的 `~/.solar` + `~/.claude` 到 `~/solar-backup-<ISO_TS>/`
- rollback 从最近一次 `~/solar-backup-*/` 还原, 如有多份按 ts 选最新
- dry-run 解到 `/tmp/migrate-dryrun-<ts>/`, 完成后自动清理
- 重复 import 同一 bundle 必须检测并提示 (通过 bundle-meta.json 的 bundle_id UUID)

### 不破坏现状
- `solar-harness` 原有子命令行为 **完全不变** (只新增 `migrate` 路由, 其他 case 分支不动)
- 本 Sprint 不动: coordinator.sh / coordinator-watchdog.sh / codex-bridge.sh 业务逻辑
- 不破坏已 passed 的 sprints: 172812/192238/203859/211820/222017/062851

### 工程
- 所有 shell 脚本 `bash -n` + `shellcheck` (如有) 通过
- tar 操作用 `tar cf` + `--exclude-from=<file>`, **禁止** `tar cvf` (v 输出会泄露路径到日志)
- 分阶段展开顺序固定: (1) 系统级 rc/plist → (2) Solar 本体 → (3) Claude 配置 → (4) secrets (需密码), 任一阶段失败立即 rollback + 非零退出
- LaunchAgents plist 展开后 `launchctl load -w <plist>`, 已有同名 agent 先 `unload` 再 `load`
- bundle 默认路径 `~/solar-bundles/solar-bundle-<hostname>-<YYYYMMDD-HHMMSS>.tar`, 可 `--out` 覆盖

### 可观测
- export/import/verify 全程 log 到 `~/.solar/harness/logs/migrate-<ts>.log`, 包含每个文件操作 + SHA256 + 错误
- import 结束后打印 doctor 报告摘要, doctor 非零则整体退出非零
- 过程中 `^C` 中断触发 cleanup: 删除半成品 bundle / 调用 rollback / 清理 /tmp/migrate-*
