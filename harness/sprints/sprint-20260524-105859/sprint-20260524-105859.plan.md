# Plan — Knowledge Ingest Dispatcher 统一控制面

## Execution Strategy

先控制风险，再恢复吞吐：

1. 保持 ThunderOMLX 全量 backfill 暂停。
2. 实现 registry + dispatcher 的最小闭环。
3. 对 raw/vault 小样本跑 JSON-first extraction。
4. pass rate 达标后恢复微批 backfill。

## DAG Nodes

| Node | Goal | Depends | Write Scope |
|---|---|---|---|
| N1 | registry schema + migrations | [] | `lib/knowledge_ingest_registry.py`, `tests/test-knowledge-ingest-registry.sh` |
