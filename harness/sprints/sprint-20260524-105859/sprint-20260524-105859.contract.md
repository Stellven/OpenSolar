# Sprint Contract — P0 Knowledge Ingest Dispatcher 统一控制面

Created: 2026-05-24T14:58:59Z
Sprint: `sprint-20260524-105859`
Project: `/Users/lisihao/Solar/harness`

## Summary

当前知识库抽取链路已能跑，但仍是多入口补丁式闭环：raw、Obsidian vault、accepted、YouTube、ChatGPT、Web、GitHub、Solar 产物各自落盘、各自触发 QMD/ThunderOMLX，导致漏抽取、状态不一致、validation_failed 失控、QMD 水位不可观测。

本 sprint 要建立统一控制面：所有来源只提交 ingest event，由 `knowledge_ingest_dispatcher` 统一状态机、幂等、重试、验证、熔断和 QMD 微批索引。QMD 继续做 deterministic indexing，ThunderOMLX 只做 L2 semantic extraction，且输出改为 JSON-first，再由 deterministic renderer 生成 `*.extracted.md`。

## Stop Rule

在 JSON-first contract、registry、validator、circuit breaker 未落地前，不允许继续全量 ThunderOMLX backfill。允许 raw/vault QMD deterministic index 继续运行。

## Requirements

### R1 — Single Dispatcher

所有来源必须通过统一 dispatcher 提交 ingest event：

- ChatGPT
- Web captures
- YouTube transcript / Tech Hotspot Radar
- GitHub trends digest
- Solar accepted/export/runtime artifacts
- Manual raw imports
- Obsidian vault canonical markdown

采集器不得直接决定最终状态，不得绕过 registry 直接宣称 DONE。

### R2 — SQLite Registry

新增 `Knowledge/_registry/knowledge_ingest.sqlite` 作为事实源，至少包含：

- `documents`
- `spans`
- `ingest_events`
- `qmd_index_events`
- `extract_jobs`
- `extract_outputs`
- `validation_results`
- `relations`
- `watermarks`

JSON manifest 只是 registry 的 sidecar/export，不再作为唯一事实源。

### R3 — Obsidian Vault First-Class Source

Obsidian 正文页必须作为 canonical knowledge，不塞进 `_raw`：

- `concepts/`
- `references/`
- `synthesis/`
- `projects/`
- `entities/`
- `rules/`
- `skills/`

Vault adapter 必须解析 frontmatter、tags、wikilinks、aliases、folder type，并生成 span sidecar。

### R4 — Span Sidecar

raw/vault 都必须生成 sidecar spans，不改写原文：

- `Knowledge/_vault_index/spans/.../*.spans.json`
- `Knowledge/_raw/.../*.spans.json` 或 registry 等价记录

span 必须保留 `doc_id`、`source_sha256`、`span_id`、`start_line`、`end_line`、`heading_path`、`text_sha256`。

### R5 — JSON-First ThunderOMLX Extraction

ThunderOMLX 不再直接生成最终 Markdown。流程必须是：

```text
source spans
  -> ThunderOMLX extracted.candidate.json
  -> JSON/schema/evidence validator
  -> optional repair_once
  -> deterministic extracted.md renderer
  -> QMD extracted index
```

模型输出 schema 至少包含：

- `summary`
- `core_facts`
- `functional_modules`
- `commands_api_config`
- `architecture`
- `risks`
- `open_questions`
- 每个 claim 的 `evidence: ["Sxxx"]`

### R6 — Validator / Repair / Quarantine

Validator 必须分层：

- syntax validator
- evidence validator
- content validator
- policy validator

失败错误码必须标准化，例如：

- `E_SCHEMA_JSON_PARSE`
- `E_SCHEMA_REQUIRED_FIELD_MISSING`
- `E_EVIDENCE_EMPTY`
- `E_EVIDENCE_UNKNOWN_SPAN`
- `E_OUTPUT_TOO_SHORT`
- `E_OUTPUT_TRUNCATED`
- `E_MODEL_REFUSAL`
- `E_SOURCE_SHA_MISMATCH`

repair 最多 1 次；二次失败必须 `QUARANTINED`，不得进入 QMD extracted index。

### R7 — Circuit Breaker

backfill 必须有熔断：

- rolling window: 25
- validation fail rate > 25% 则暂停 extract queue
- consecutive validation failures >= 5 则暂停 extract queue
- 暂停后只允许 repair/sample/manual resume

### R8 — QMD Microbatch + Watermarks

QMD 必须分层记录水位：

- raw
- vault
- extracted

必须支持：

- flush_every_docs
- flush_every_seconds
- pending jobs
- failed jobs
- last successful batch

不能只显示一个全局 “18h ago”。

### R9 — Query Grounding Policy

默认查询使用 extracted 做 semantic routing，但最终事实必须回源 raw/vault/source spans：

- extracted 命中没有 source span 时只能作为 hint
- final context builder 必须优先 raw/vault grounding
- derived artifact 不得单独作为事实终点

### R10 — Historical Backfill Provenance

历史 raw/vault/extracted 回填必须标注 provenance quality：

- `complete`
- `inferred`
- `observed`
- `unknown`

不得伪造旧模型、旧 prompt、旧 usage。

## Scope

### In Scope

- Dispatcher CLI and registry.
- Raw/vault discovery and event submission.
- Span sidecar generation.
- JSON-first ThunderOMLX extraction contract.
- Deterministic Markdown renderer.
- Validator, repair, quarantine.
- Circuit breaker.
- QMD microbatch watermarks.
- Health/report commands.
- Small-sample migration and validation.

### Out of Scope

- Replacing QMD embedding model.
- Replacing ThunderOMLX backend.
- Full historical 9807 raw + 1349 vault backfill before sample pass.
- Rewriting Obsidian source notes.
- Making RAGFlow the source of truth.

## Red Flags

- Any adapter directly marks a document DONE without registry.
- ThunderOMLX output Markdown enters QMD without JSON validation.
- Validation failed artifacts enter default QMD extracted index.
- Obsidian vault pages are treated as raw imports.
- Backfill continues after circuit breaker trips.
