## 1. Problem (Restated)

Solar 知识库现状已有：raw 落盘 (`Knowledge/_raw`)、QMD deterministic index (`qmd_adapter.py` / `qmd-embed-runner.sh`)、Obsidian vault (`obsidian-vault-indexer.py`)、ThunderOMLX 抽取 (`knowledge-semantic-extract.py`)、Tech Hotspot / AI Influence / ChatGPT / Apple Notes 多入口 (`chatgpt-conversation-ingest.py`、`apple_notes_ingest.py`、`mineru_extract.py`、wiki-upload-* 系列)。

问题：**入口、状态、契约三层都没统一**：
1. 入口分散：raw、vault、accepted、YouTube、ChatGPT、Web、GitHub、Solar 产物各走各的脚本。
2. 状态不一致：manifest / log / report / QMD 互相对不上账。
3. 抽取无强契约：ThunderOMLX 直接出 Markdown → `validation_failed` 大面积出现。

本 sprint 建立统一控制面 `knowledge_ingest_dispatcher` + SQLite registry + JSON-first extraction + 分层 validator + circuit breaker + QMD 三层水位。

### 3.2 State Machine (PRD F4)

```text
NEW
 └► SOURCE_CAPTURED ──► RAW_MATERIALIZED ──► RAW_OR_VAULT_QMD_INDEX_PENDING ──► RAW_OR_VAULT_QMD_INDEXED
                                  │                                                       │
                                  └► VAULT_DISCOVERED ──► SPAN_MAPPED ───────────────────►┤
                                                                                          ▼
                                                                                EXTRACT_ELIGIBLE
                                                                                          │
                                                                                          ▼
                                                            THUNDEROMLX_EXTRACT_PENDING ──► THUNDEROMLX_EXTRACT_RUNNING
                                                                                          │
                                                                                          ▼
                                                                           EXTRACT_CANDIDATE_GENERATED
                                                                                          │
                                                                                          ▼
                                                                           EXTRACT_VALIDATION_PENDING
                                                                          ┌───────────────┴──────────────┐
                                                                          ▼                              ▼
                                                                EXTRACTED_VALIDATED              VALIDATION_FAILED
                                                                          │                              │
                                                                          ▼                              ▼
                                                              EXTRACTED_QMD_INDEX_PENDING        REPAIR_PENDING (1x)
                                                                          │                              │
                                                                          ▼                              ▼ fail again
                                                                EXTRACTED_QMD_INDEXED                QUARANTINED
                                                                          │
                                                                          ▼
                                                                          DONE
```

**Invariants**:
- 状态机驱动只在 dispatcher (N2) 里；adapter 只 submit event，不直接置状态。
- `QUARANTINED` 与 `VALIDATION_FAILED` 永不进入 `EXTRACTED_QMD_INDEX_PENDING` —— 由 N5 validator + N6 indexer 共同强制。
- 任何 reverse transition（DONE → 任何状态）必须经 `wiki-reingest-quarantine` 显式 reset，且 registry 写 audit 行。

### 3.3 Registry Tables (PRD R2)

```sql
-- N1 Acceptance: documents / spans / extract_jobs / validation_results / watermarks 必存在；migration 幂等

CREATE TABLE IF NOT EXISTS documents (
  doc_id TEXT PRIMARY KEY,                -- sha256(source_kind || source_path) prefix 16
  source_kind TEXT NOT NULL,              -- raw_chatgpt | obsidian_vault | accepted | youtube ...
  source_path TEXT NOT NULL,
  source_adapter TEXT NOT NULL,
  content_kind TEXT NOT NULL,             -- markdown | json | html | pdf | transcript
  declared_doc_type TEXT,                 -- concept | reference | synthesis | project | entity | rule | skill
  source_sha256 TEXT NOT NULL,            -- of raw bytes
  current_state TEXT NOT NULL,            -- state machine cursor
  ingest_policy TEXT NOT NULL DEFAULT 'default',
  extract_policy TEXT NOT NULL DEFAULT 'default',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  provenance_quality TEXT NOT NULL DEFAULT 'observed', -- complete|inferred|observed|unknown (R10)
  UNIQUE (source_kind, source_path)
);

CREATE TABLE IF NOT EXISTS spans (
  span_id TEXT PRIMARY KEY,               -- doc_id || ':' || sequence
  doc_id TEXT NOT NULL REFERENCES documents(doc_id),
  start_line INTEGER NOT NULL,
  end_line INTEGER NOT NULL,
  heading_path TEXT,                      -- "Section A > Sub 1"
  text_sha256 TEXT NOT NULL,
  source_sha256 TEXT NOT NULL,            -- of the parent doc; for source-mismatch check
  byte_offset_start INTEGER,
  byte_offset_end INTEGER,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS ingest_events (
  event_id TEXT PRIMARY KEY,              -- evt_<uuid7> by adapter; dispatcher dedupe
  doc_id TEXT NOT NULL REFERENCES documents(doc_id),
  event_kind TEXT NOT NULL,               -- submit | state_change | retry | quarantine | reset
  from_state TEXT,
  to_state TEXT,
  source_adapter TEXT NOT NULL,
  payload_json TEXT NOT NULL,
  ts TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS qmd_index_events (
  event_id TEXT PRIMARY KEY,
  doc_id TEXT NOT NULL REFERENCES documents(doc_id),
  layer TEXT NOT NULL,                    -- raw | vault | extracted
  qmd_batch_id TEXT,
  qmd_status TEXT NOT NULL,               -- pending | indexed | failed
  retry_count INTEGER NOT NULL DEFAULT 0,
  ts TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS extract_jobs (
  job_id TEXT PRIMARY KEY,
  doc_id TEXT NOT NULL REFERENCES documents(doc_id),
  source_span_ids TEXT NOT NULL,          -- JSON array
  prompt_template_id TEXT NOT NULL,
  model TEXT NOT NULL,
  state TEXT NOT NULL,                    -- pending | running | candidate | validated | failed | repaired | quarantined
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  repair_count INTEGER NOT NULL DEFAULT 0
);

CREATE TABLE IF NOT EXISTS extract_outputs (
  output_id TEXT PRIMARY KEY,
  job_id TEXT NOT NULL REFERENCES extract_jobs(job_id),
  kind TEXT NOT NULL,                     -- candidate_json | rendered_md
  path TEXT NOT NULL,
  sha256 TEXT NOT NULL,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS validation_results (
  result_id TEXT PRIMARY KEY,
  job_id TEXT NOT NULL REFERENCES extract_jobs(job_id),
  layer TEXT NOT NULL,                    -- syntax | evidence | content | policy
  passed INTEGER NOT NULL,                -- 0|1
  error_code TEXT,                        -- E_SCHEMA_JSON_PARSE ... (R6)
  detail_json TEXT,
  ts TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS relations (
  relation_id TEXT PRIMARY KEY,
  source_doc_id TEXT NOT NULL,
  target_doc_id TEXT NOT NULL,
  kind TEXT NOT NULL,                     -- wikilink | tag_cooccurrence | alias_resolve | extract_source
  weight REAL NOT NULL DEFAULT 1.0,
  ts TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS watermarks (
  layer TEXT PRIMARY KEY,                 -- raw | vault | extracted
  last_indexed_ts TEXT,
  pending_count INTEGER NOT NULL DEFAULT 0,
  failed_count INTEGER NOT NULL DEFAULT 0,
  last_batch_id TEXT,
  last_batch_ts TEXT,
  updated_at TEXT NOT NULL
);

-- Migration idempotency: 用 IF NOT EXISTS + ALTER TABLE 守门；migration_log table 记录 schema_version
CREATE TABLE IF NOT EXISTS migration_log (
  version INTEGER PRIMARY KEY,
  applied_at TEXT NOT NULL,
  checksum TEXT NOT NULL
);
```

### 3.4 Ingest Event Schema (PRD F1)

```jsonc
{
  "event_id":         "evt_01HXXXXXXX",     // adapter generates uuid7
  "source_kind":      "obsidian_vault",     // enum (see documents.source_kind)
  "source_path":      "/Users/lisihao/Knowledge/concepts/qmd-indexing.md",
  "source_adapter":   "obsidian_adapter",
  "content_kind":     "markdown",
  "declared_doc_type":"concept",
  "ingest_policy":    "default",
  "extract_policy":   "default",
  "extra": { /* adapter-specific */ }
}
```

Dispatcher checksum: `dedupe_key = sha256(source_kind || ':' || source_path || ':' || source_sha256)`；重复提交直接 ack 老 doc_id。

### 3.5 JSON-First Extraction Contract (PRD R5, F5)

`extracted.candidate.json` minimal schema:

```jsonc
{
  "doc_id":              "...",
  "source_span_ids":     ["S001","S007","S012"],
  "summary":             "...",
  "core_facts":          [{"text":"...", "evidence":["S001"]}],
  "functional_modules":  [{"name":"...", "evidence":["S007"]}],
  "commands_api_config": [{"shell":"...", "evidence":["S012"]}],
  "architecture":        {"text":"...", "evidence":["S001","S007"]},
  "risks":               [{"text":"...", "evidence":["S012"]}],
  "open_questions":      [{"text":"...", "evidence":[]}],
  "extractor": {
    "model":             "thunderomlx-...",
    "prompt_template_id":"v1",
    "ts":                "..."
  }
}
```

- 每个 claim 必须有 `evidence: [span_id, ...]`。Validator (N5) 检查每个 span_id 都存在于 `spans` 表且 `source_sha256` 与父 doc 一致；不一致触发 `E_SOURCE_SHA_MISMATCH`。
- Renderer (N4) 是 deterministic：input candidate JSON → output `*.extracted.md` with frontmatter (doc_id, source_kind, source_path, extracted_at, evidence_index)；同一 candidate 多次渲染产物字节相同。

### 3.6 Validator Error Codes (PRD R6)

| 代码 | layer | 触发条件 |
|---|---|---|
| `E_SCHEMA_JSON_PARSE` | syntax | candidate 不是合法 JSON |
| `E_SCHEMA_REQUIRED_FIELD_MISSING` | syntax | summary/core_facts/evidence 等必填缺失 |
| `E_EVIDENCE_EMPTY` | evidence | claim 的 evidence 数组为空 |
| `E_EVIDENCE_UNKNOWN_SPAN` | evidence | evidence 引用了 spans 表外的 span_id |
| `E_OUTPUT_TOO_SHORT` | content | 渲染后 markdown < threshold（如 < 200 chars 且非占位）|
| `E_OUTPUT_TRUNCATED` | content | 模型输出以 `...` 或 `[truncated]` 收尾 |
| `E_MODEL_REFUSAL` | content | 模型返回拒答标记 |
| `E_SOURCE_SHA_MISMATCH` | policy | evidence span 的 source_sha256 与当前 doc 不一致（说明原文已改）|

Repair flow: 第一次 fail → `state=REPAIR_PENDING`，dispatcher 用 "fix-only" prompt 重抽一次 → 仍 fail → `state=QUARANTINED`，永不入 extracted QMD index。

### 3.7 Circuit Breaker (PRD R7)

```python
# N7 implementation sketch
WINDOW = 25
fails = recent_validation_results(window=WINDOW)
rate = sum(1 for r in fails if not r.passed) / max(len(fails), 1)
consec = consecutive_failures_tail(fails)
paused = (len(fails) >= WINDOW and rate > 0.25) or consec >= 5
if paused:
    set_extract_queue_state("paused")  # raw/vault QMD index 不受影响
```

- 暂停只针对 `EXTRACT_ELIGIBLE` → `THUNDEROMLX_EXTRACT_PENDING` 这一跳。
- raw/vault QMD index pipeline 是独立 worker，不消费 extract queue → 不受影响。
- 手动恢复：`solar-harness wiki knowledge-ingest circuit-breaker resume --confirm`。

### 3.8 QMD Microbatch + Watermarks (PRD R8, F8)

`watermarks` 表每层一行（raw / vault / extracted）。`N6_qmd_microbatch`：
- 配置：`flush_every_docs` 默认 50，`flush_every_seconds` 默认 30。
- 每次 flush 后更新该 layer 的 `last_indexed_ts`、`last_batch_id`、`last_batch_ts`、`pending_count`、`failed_count`。
- `qmd-watermarks --json` 输出三层，不再合并为 "全局 18h ago"。

### 3.9 Query Grounding (PRD R9)

由 **N9 (out of scope this sprint)** 在 `lib/solar-knowledge-context.py` 加 hook：
- extracted 命中 → 必须从 registry 查 `relations.kind=extract_source` 回到 raw/vault span
- 找不到 span → extracted 仅作 hint，不进 final evidence
- final context builder 优先 raw/vault grounding

本 sprint 不实现 N9；但 N1 registry 已为它预留 `relations` 表与 `extract_source` kind。

## 6. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Registry migration 在 partial-fail 下脏库 | `migration_log` + 事务包裹；幂等检查 + 失败 rollback；增加 `--dry-run` flag |
| ThunderOMLX JSON 输出仍不稳 | small schema + few-shot exemplars in prompt template；`repair_once` 用更严格的 fix-only prompt；circuit breaker 兜底 |
| span_id 漂移（原文被改） | `source_sha256` 校验；E_SOURCE_SHA_MISMATCH 触发 reingest；不静默通过 |
| 重复 ingest event 风暴 | dedupe by `(source_kind, source_path, source_sha256)`；adapter 端缓存 last_seen_sha |
| QMD 微批与现有 `qmd-embed-runner.sh` 抢锁 | N6 通过 `qmd_adapter.py` 调用而非直接调 runner；现有 file-lock 协议复用 |
| backfill 暂停后 raw/vault QMD index 也卡 | 单独 worker 化；N7 暂停信号只挂到 extract queue 的 producer，不挂到 QMD indexer |
| D9 PM 期望落地但未在本 sprint 闭环 | 在 PRD/handoff/eval 都明示 "deferred to follow-up sprint"；不偷偷绕过 |

## 7. Stop Rules (Hard Gates)

- 任何 D1-D8 + D10 的 verify 命令 FAIL → 不进 DONE
- registry migration 不幂等 → N1 不算 done
- ThunderOMLX 输出未经 N5 validator 直接进 QMD → 视为契约违反，N6 必须拒收
- backfill 在 circuit breaker tripped 状态下继续跑 → P0 incident，立即 rollback
- 改 PRD `Out of Scope` 列出的任一项 → 违例
- 缺 verifier 决策 → 不进 DONE
