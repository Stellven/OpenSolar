## Requirements

P0 修复协调器 dispatch_to_pane 同 pane 连派覆盖 bug — 当 30s 内连派 2 个 dispatch 给同一 builder pane, 后派的 send-keys + 预解锁 C-u 会清空前一个未提交的 input box 文本, 导致前一个 sprint 永远丢失。架构师 investigation-002 找到 3 个叠加漏洞 (无 per-pane 占用记录 / C-u 破坏性预解锁 / verify 双命中可被旧任务残留欺骗), 修复方案 P0+P1: per-pane PANE_CURRENT_SPRINT tracking + dispatch busy 守卫 (return code 2) + flock per-pane lock + handle_passed/failed 末尾 clear assignment + 持久化到 .pane-assignments. 详见 architect-design-dispatch-fix-done.md 和 investigation-002.md
