## 关键设计决策

### D4: mkdir 原子锁替代 flock
合约原方案用 `flock -n`, 但 macOS 无 flock (which flock = not found)。
sprint-20260503-111139 已验证 mkdir 原子锁在 macOS 可用。
实现: `mkdir "$lock_dir" 2>/dev/null` 拿锁, `rm -rf "$lock_dir"` 释放。
进程异常退出 OS 不自动释放 mkdir 锁 (与 flock 不同), 但:
1. D2 busy 守卫提供 30min 超时强制重派
2. lock 内写 pid, stale 检测可后续优化

### D6: rollback_state_cache
return 2 时不调 save_state — 但 save_state 在 dispatch 之前已执行 (line 1924)。
需要回滚 state cache, 否则下轮 check_state_changed 认为已处理。
rollback_state_cache 从 .coordinator-state 中删除目标 sid 条目。

## Known Risks
1. mkdir 锁 stale: 进程 crash 后锁目录残留, 需 D2 超时兜底
2. rollback_state_cache python3 静默失败: 使用 `|| true`, 不阻塞主流程
3. DISPATCH_MOCK 环境变量: 仅测试时设置, 生产路径无此变量
