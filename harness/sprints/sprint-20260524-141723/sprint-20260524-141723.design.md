## 2. Architecture Decisions

### A1: File Naming — `*.semantic.md` (正名) + `*.extracted.md` (symlink 兼容)

**决断**: 采用 intake 要求的 `*.semantic.md`；105859 的 `*.extracted.md` 保留为 symlink 别名。

**理由**:
- intake contract 明确要求 `semantic.md`
- `extracted.md` 只在 105859 内部 renderer 使用，外部无硬依赖
- symlink 兼容期 2 sprint，不破坏 grep 链路

**实施** (S03 builder):
1. `knowledge_extracted_renderer.py` 输出文件名改为 `*.semantic.md`
2. renderer 写入 `.semantic.md` 后创建 `*.extracted.md → *.semantic.md` symlink
3. `qmd_index_events.layer` 新写入用 `semantic`，旧数据不动
4. `watermarks` 表: DELETE `extracted` 行, INSERT `semantic` 行

**Rollback**: 改回 `extracted.md`，移除 symlink 逻辑，只需改 renderer 一处。

### A2: Schema Extension — In-Place Add Column (migration v2)

**决断**: `ALTER TABLE ADD COLUMN`，不走 schema v2 新表。

**理由**:
- 105859 数据量极小 (< 50 rows)，无 migration 性能问题
- SQLite `ALTER TABLE ADD COLUMN` 原子操作，向后兼容
- 避免新表 join 开销

**新增列**:
```sql
ALTER TABLE extract_jobs ADD COLUMN schema_version TEXT NOT NULL DEFAULT 'v1';
ALTER TABLE extract_jobs ADD COLUMN endpoint TEXT;
ALTER TABLE extract_jobs ADD COLUMN started_at TEXT;
ALTER TABLE extract_jobs ADD COLUMN finished_at TEXT;

ALTER TABLE extract_outputs ADD COLUMN latency_ms REAL;
ALTER TABLE extract_outputs ADD COLUMN token_input_count INTEGER;
ALTER TABLE extract_outputs ADD COLUMN token_output_count INTEGER;
ALTER TABLE extract_outputs ADD COLUMN cost_estimate REAL;
ALTER TABLE extract_outputs ADD COLUMN model_fingerprint TEXT;
```

- `migration_log` 记录 version=2
- 幂等: try/except 跳过已存在列

### A3: State Machine Extension — 2 个新独立状态

**决断**: 新增 `EXTRACT_FAILED_RETRYABLE` 和 `DONE_RAW_ONLY_WARN` 为独立状态，不合并到 `VALIDATION_FAILED`。

**理由**:
- `EXTRACT_FAILED_RETRYABLE`: 抽取阶段 (ThunderOMLX call) 失败但可重试 — 与 `VALIDATION_FAILED` (抽取成功但校验失败) 语义不同
- `DONE_RAW_ONLY_WARN`: 终态，raw/vault 可查但无 semantic — circuit breaker 暂停或 adapter 不支持抽取时使用
- 合并会导致 retry 逻辑混乱

**完整状态机** (扩展自 105859 design):

```text
NEW
 └► SOURCE_CAPTURED ──► RAW_MATERIALIZED ──► RAW_OR_VAULT_QMD_INDEX_PENDING
                                  │                       │
                                  └► VAULT_DISCOVERED     │
                                          │               │
                                          ▼               ▼
                                      SPAN_MAPPED ──► RAW_OR_VAULT_QMD_INDEXED
                                                          │
                                                          ▼
                                                  EXTRACT_ELIGIBLE
                                                     ┌──────┼──────────────┐
                                                     │      │              │
                                                     ▼      │              ▼
                                          (skip extract)   │   THUNDEROMLX_EXTRACT_PENDING
                                                     │      │              │
                                                     ▼      │              ▼
                                          DONE_RAW_ONLY_WARN    THUNDEROMLX_EXTRACT_RUNNING
                                                                   ┌──────┴──────┐
                                                                   │             │
                                                                   ▼             ▼
                                                    EXTRACT_CANDIDATE_     EXTRACT_FAILED_
                                                    GENERATED              RETRYABLE
                                                                   │             │
                                                                   │             │ (retry max 3)
                                                                   ▼             │
                                                    EXTRACT_VALIDATION_   │
                                                    PENDING               │
                                                      ┌──┴──┐            │
                                                      ▼     ▼            │
                                          EXTRACTED_    VALIDATION_   │
                                          VALIDATED     FAILED        │
                                               │             │        │
                                               ▼             ▼        │
                                    EXTRACTED_QMD_   REPAIR_PENDING    │
                                    INDEX_PENDING    (1x)              │
                                               │           │            │
                                               ▼           ▼ fail again │
                                    EXTRACTED_QMD_  QUARANTINED        │
                                    INDEXED                             │
                                               │                        │
                                               ▼                        │
                                             DONE ◄─────────────────────┘
                                              (retry exhausted → DONE_RAW_ONLY_WARN)
```

**新转移规则**:

| From | To | Trigger |
|---|---|---|
| EXTRACT_ELIGIBLE | DONE_RAW_ONLY_WARN | extract_policy=skip OR circuit breaker paused AND bypass window expired |
| THUNDEROMLX_EXTRACT_RUNNING | EXTRACT_FAILED_RETRYABLE | ThunderOMLX call exception |
| EXTRACT_FAILED_RETRYABLE | THUNDEROMLX_EXTRACT_RUNNING | retry (max 3, exponential backoff) |
| EXTRACT_FAILED_RETRYABLE | DONE_RAW_ONLY_WARN | max retry exceeded |

**兼容**: 现有 documents 行无新状态值；dispatcher 读到未知状态保持不动。

### A5: Query Grounding Interface Contract (O10)

**决断**: 定义 GroundingHook 只读接口，实现留给 S04。

**接口**:

```
GroundingHook:
  input:
    query: str
    extracted_hits: [{doc_id, score, candidate_json_path}]
    registry: knowledge_ingest_registry connection

  output:
    grounded_context: [{
      claim_text: str,
      evidence_spans: [{span_id, source_path, text_excerpt}],
      confidence: float,          # 0.0-1.0
      source_layer: "semantic" | "raw" | "vault"
    }]

  fallback policy:
    - extracted hit 无 span 引用 → source_layer=semantic, confidence=0.5
    - span 在 registry 不存在 → drop 该 claim, log warning
    - raw/vault 同 source_path → 优先用 raw/vault 文本 grounding
    - 超时 2s → fallback raw/vault only, log warning
```

**Consumer**:
1. `solar-knowledge-context.py` — context inject 优先 extracted 但 grounding 回 raw/vault
2. Tech Hotspot report — reader 优先 `*.semantic.md`，fallback 到 transcript/raw
3. status server — dashboard 展示 grounding 覆盖率

**约束**:
- 只读操作，不修改 registry 或文件
- 超时不阻塞查询路径

## 3. System Layering

```text
Layer 0: Source Adapters (8 kinds)
  职责: discover + submit ingest_event
  依赖: dispatcher API
  降级: 单 adapter 失败不阻塞其他

Layer 1: Dispatcher (state machine driver)
  职责: idempotency, state transition, queue drain
  依赖: Registry (SQLite)
  新增: EXTRACT_FAILED_RETRYABLE retry loop + DONE_RAW_ONLY_WARN
  降级: crash 后 restart 从 registry 恢复

Layer 2: Registry (SQLite, source of truth)
  职责: documents/spans/events/jobs/outputs/validation/relations/watermarks
  新增: extract_jobs (schema_version, endpoint, timestamps)
        extract_outputs (latency_ms, tokens, cost, model_fingerprint)
        watermarks layer: extracted → semantic
  降级: WAL mode crash-safe; 锁 → retry 3 次

Layer 3: Extract Pipeline (JSON-first → Renderer → Validator)
  职责: ThunderOMLX extraction + deterministic rendering + validation
  新增: schema_version 跟踪, model_fingerprint
  降级: extraction fail → DONE_RAW_ONLY_WARN; validation fail → quarantine

Layer 4: QMD Indexer (3 watermarks: raw/vault/semantic)
  职责: microbatch flush, watermark tracking
  新增: layer name extracted → semantic
  降级: indexer down → queue 排队, 恢复后 flush

Layer 5: Query & Reporting
  职责: GroundingHook (A5) + report reader + context inject
  新增: A5 grounding interface
  降级: hook 超时 → raw/vault only

Layer 6: Observability & Health
  职责: circuit breaker + health CLI + dashboard UI
  新增: retry count metrics, DONE_RAW_ONLY_WARN count
  降级: circuit breaker trips → raw/vault QMD 照常
```

## 4. Interface Contracts

### 4.1 Dispatcher ↔ Registry

| Method | Input | Output | Invariant |
|---|---|---|---|
| `submit_event(event)` | IngestEvent | `{ok, doc_id}` | idempotent by (source_kind, source_path, source_sha256) |
| `transition(doc_id, to_state)` | doc_id, target_state | `{ok, from_state, to_state}` | only valid transitions (§A3) |
| `drain_queue(state, limit)` | state, batch_size | `[doc_id]` | FIFO by updated_at |

### 4.2 Adapter ↔ Dispatcher

| Method | Input | Output |
|---|---|---|
| `discover(limit)` | max docs | `[IngestEvent]` |
| `verify_spans(doc_id)` | doc_id | `{span_count, verified, mismatch}` |

### 4.3 Extractor ↔ Registry

| Method | Input | Output |
|---|---|---|
| `create_job(doc_id, span_ids, prompt_id, model)` | params | `{job_id}` |
| `write_output(job_id, kind, path, sha256, latency_ms, tokens, cost, model_fp)` | data | `{output_id}` |
| `update_job_state(job_id, state)` | state | `{ok}` |

### 4.4 Validator ↔ Registry

| Method | Input | Output |
|---|---|---|
| `validate(job_id)` | job_id | `{passed, errors[]}` |
| `write_validation(result)` | ValidationResult | `{result_id}` |

### 4.5 GroundingHook ↔ Registry (Read-only)

| Method | Input | Output |
|---|---|---|
| `resolve_spans(span_ids)` | `[span_id]` | `[{span_id, source_path, text_excerpt}]` |
| `get_relations(doc_id, kind)` | doc_id, relation_kind | `[{target_doc_id, weight}]` |

## 5. Failure Recovery

| 故障 | 恢复 | 实现 |
|---|---|---|
| Dispatcher crash mid-transition | Registry audit trail → restart 重跑 drain | ingest_events 表 |
| Extractor crash | EXTRACT_FAILED_RETRYABLE → retry max 3 | dispatcher retry loop |
| Validator crash | Job stays EXTRACT_VALIDATION_PENDING → next drain re-validates | idempotent validate |
| QMD indexer crash | qmd_status=failed → retry next microbatch | retry_count |
| Full pipeline pause (circuit breaker) | raw/vault QMD continues; extract only paused | N7 health |
| Retry exhausted | DONE_RAW_ONLY_WARN terminal; raw/vault searchable | state machine |
| Source changed after extraction | E_SOURCE_SHA_MISMATCH → quarantine → reingest | validator |
