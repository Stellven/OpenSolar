## Done 定义达成

### D1: 根因定位 ✅

诊断报告: `~/.solar/reports/2026-05-02-watchdog-respawn-fail.md`

**根因**: tmux `respawn-pane` 不继承用户 shell profile (`~/.zshrc`),新进程的 PATH 只包含 `/usr/bin:/bin:/usr/sbin:/sbin`。`claude` 安装在 `~/n/bin/claude`,不在极简 PATH 中 → `claude: command not found` → exit 127 → "Pane is dead (status 127)"。

**证据**:
1. `tmux show-environment -t solar-harness | grep PATH` → 空 (tmux session 无 PATH)
2. `which claude` → `/Users/lisihao/n/bin/claude`
3. `env -i PATH="/usr/bin:/bin" bash -c 'which claude'` → 找不到
4. `env -i HOME="$HOME" /opt/homebrew/bin/bash -c '(PATH保底代码) which claude'` → `/Users/lisihao/.local/bin/claude` (找到!)

报告包含 4 个排查假设 (H1-H4),每个的验证方法和真实证据。

### D2: Watchdog respawn 修复 ✅

**改动 1** — `coordinator-watchdog.sh` respawn 命令:

- 新: `env HOME='${HOME}' PATH='${_user_path}' /opt/homebrew/bin/bash ${_esc_h}/start-incarnation.sh ...`
