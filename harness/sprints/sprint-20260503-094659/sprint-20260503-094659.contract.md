# Execution Contract Template

日期: 2026-05-03
任务名: V2 Phase A — Control Plane 收敛的第一个最小可执行 sprint
执行方: solar-harness

## 1. Objective

- 在不重写整套 harness 的前提下，引入一个正式的 `control-plane.sh` 入口和统一 state helper，并让至少一条真实状态推进路径接入该 helper，作为后续 Phase A 收敛的基座。

## 2. Background

- 上游 research 结论：
  - v1 已完成启动链、secrets、worktree、watchdog 等基础修复
  - 当前 Phase A 不应重复这些工作
  - 当前最缺的是“正式控制面入口”和“统一状态读写接口”
- 这一单只做最小闭环，不做全量迁移

## 3. Allowed Scope

- 允许修改:
  - `~/.solar/harness/solar-harness.sh`
  - `~/.solar/harness/session.sh`
  - `~/.solar/harness/lib/*.sh`
  - `~/.solar/harness/test-*.sh`
  - `~/.solar/harness/tests/*`
- 允许新增:
  - `~/.solar/harness/control-plane.sh`
  - `~/.solar/harness/lib/run-state.sh` 或等价 helper
  - `~/.solar/harness/docs/*.md`

## 4. Forbidden Actions

- 禁止改动启动链统一、secret 隔离、worktree、watchdog 的既有行为
- 禁止扩大成“全量 Phase A 重构”
- 禁止静默删除现有 sprint 生命周期能力
- 禁止只新增脚手架文件而不接入真实路径
- 禁止只改文档不改行为

## 5. Required Work

1. 新增正式入口 `control-plane.sh`，至少支持可发现的帮助输出和一个状态相关子命令
2. 新增统一 state helper，用于封装 sprint 状态读写或状态推进
3. 选择一条真实状态推进路径接入该 helper
   - 优先建议：`plan-verdict`、`eval-verdict`、`handoff-submit` 三者之一
4. 补一个最小回归验证，证明 control plane 入口和 helper 不是空壳

1. 改动文件列表
2. 执行命令列表
3. control plane help 输出或等价证据

```bash

rg -n 'run-state.sh|source .*run-state|\. .*run-state' ~/.solar/harness
rg -n 'control-plane|run-state|state helper' ~/.solar/harness/test-*.sh ~/.solar/harness/tests
```

## 9. Handoff Format

必须按以下顺序输出:

1. Summary
2. Changed Files
3. Commands Run
4. Verification Evidence
5. Known Risks
6. Not Done

特别补充:

- `Verification Evidence` 必须明确指出“哪一条真实状态路径已接入 helper”
- `Not Done` 必须明确列出“这单没有迁移哪些状态写入点”

## 10. Failure Rules

遇到以下情况必须停止并上报:

- 需要修改范围外大面积协调器逻辑
- 发现 `control-plane.sh` 无法在当前 harness 布局中以最小代价接入
- 发现真实状态路径接入 helper 会破坏当前活跃 sprint
- 测试无法运行且无合理替代验证
