## 根因定位

**真正根因**: `coordinator.sh` 主循环的 mtime 提前退出 (line 1796 `continue`) 导致 hot-reload 检查 (line 1912) **永远不可达**。

1791:   if which fswatch:        ← fswatch 未安装
1794:     sleep 10               ← 走这条路径

### 2. `~/.solar/harness/coordinator.sh` — hot-reload 边界兜底 (新增代码)

**改动位置**: line 1910-1920 (hot-reload 检查块)

**改动内容**:
- md5 命令不存在时优雅降级 (当前已有 `|| echo 'unknown'`，但需要确保 unknown==unknown 不误触发)
- exec 失败时写 PLANNER-INBOX 告警 + 不死循环
- 增加重启次数限制 (例如每 5 分钟最多 1 次)

```bash
# 增强版 hot-reload 检查
local hot_reload_tick=${HOT_RELOAD_TICK_OVERRIDE:-60}
if (( loop_count % hot_reload_tick == 0 )); then
  local current_md5
  current_md5=$(md5 -q "$HARNESS_DIR/coordinator.sh" 2>/dev/null || md5sum "$HARNESS_DIR/coordinator.sh" 2>/dev/null | cut -d' ' -f1 || echo 'unknown')

  # 兜底: md5 全失败时降级,不崩溃
  if [[ "$current_md5" == "unknown" ]]; then
    log "[hot-reload] WARNING: md5 command unavailable, skipping self-check"
    return  # 或 continue
  fi

  if [[ "$current_md5" != "$INIT_MD5" ]]; then
    log "[hot-reload] md5 changed: ${INIT_MD5} → ${current_md5}, exec restart"

    # 兜底: exec 失败时告警 + 不死循环
    if ! clean_my_pidfile || ! exec /opt/homebrew/bin/bash "$0" "$@"; then
      log "[HOT-RELOAD-FAILED] exec restart failed at $(date -u +%Y-%m-%dT%H:%M:%SZ)"
      echo "- [ ] [HOT-RELOAD-FAILED] exec restart failed at $(date -u +%Y-%m-%dT%H:%M:%SZ)" >> "$HARNESS_DIR/PLANNER-INBOX.md"
      echo "[HOT-RELOAD-FAILED]" > "$HARNESS_DIR/.planner-last-notice"
      # 更新 INIT_MD5 防止反复尝试
      INIT_MD5="$current_md5"
    fi
  fi
fi
```

## 技术方案

### 核心修复: 消除 mtime continue 导致的代码不可达

**问题**: `continue` 跳过了主循环下半部分所有周期性代码

**方案**: 用 `skip_sprint` 标志替代 `continue`

```
修复前控制流:
  loop_count++ → mtime check → [无变化] → sleep → continue → 回到 loop_count++
  (周期性检查永远不可达)

修复后控制流:
  loop_count++ → mtime check → [无变化] → sleep → skip_sprint=1
  → sprint处理 (skip_sprint=1 → 跳过)
  → 周期性检查 (始终执行) ← hot-reload 在这里
  → sleep → 回到 loop_count++
```

### 边界兜底设计

1. **md5 命令缺失**: `|| echo 'unknown'` 已有，但需加 `unknown != unknown` 守卫 (其实两个 unknown 相等不会触发，安全)
2. **exec 失败**: 用 `if ! exec ...; then` 包裹，失败时写告警 + 更新 INIT_MD5 防止死循环
3. **热重载代码被删**: hot-reload 代码在函数内的 `while true` 循环里，如果整段被注释，bash 不会报错，只是功能消失。这个无法防御，依赖代码审查。

## 风险点

| # | 风险 | 影响 | 缓解 |
|---|------|------|------|
| R1 | 去掉 continue 后，mtime 无变化时仍然解析 sprint 文件 | 无性能影响 (skip_sprint=1 跳过解析) | 用标志位而非直接去 continue |
| R2 | exec 重启后环境变量丢失 | 低风险 (exec 保持同一进程，环境变量不变) | 保持 exec 方案不变 |
| R3 | HOT_RELOAD_TICK_OVERRIDE 设太小导致频繁 md5 计算 | 极低 (md5 1 个 2000 行文件 < 1ms) | 默认 60 不变 |
| R4 | 测试脚本修改了生产 coordinator.sh | 测试误操作 | trap EXIT 还原 + 备份机制 |
| R5 | 修复后 fswatch 路径行为变化 | 无变化 (fswatch 只影响 sleep 等待方式) | 不改 fswatch 逻辑 |
