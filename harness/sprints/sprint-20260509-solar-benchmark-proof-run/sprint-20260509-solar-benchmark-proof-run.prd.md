# PRD — Solar Benchmark Proof Run

## Problem

The previous benchmark results were generated by local commands. The user needs proof that Solar-Harness itself can dispatch a real pane to execute and report the benchmark.

## User Outcome

The user can inspect:

- `events.jsonl` showing wake/dispatch into a real pane.
- tmux pane output showing benchmark commands executed by the builder persona.
- a handoff file written by the builder with command results.
- generated benchmark reports and evidence bundles.

## Requirements

- Use the real main builder pane `solar-harness:0.2`.
- Run capability and platform workflow benchmarks.
- Run `solar-harness verify-integrations`.
- Preserve raw evidence and report paths.

## Non Goals

- Do not run heavy 24GB migration export/import.
- Do not mutate product code.
- Do not claim token usage if only shell commands ran.

## Acceptance

- A handoff file exists at `sprint-20260509-solar-benchmark-proof-run.handoff.md`.
- The handoff includes command outputs and report paths.
- `events.jsonl` records pane dispatch/wake evidence.
