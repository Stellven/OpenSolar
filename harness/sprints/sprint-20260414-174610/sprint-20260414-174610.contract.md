## 约束

- 复用现有技能，不重新实现已有功能 (StatsPAI/pyfixest/marginaleffects/academic-paper-composer/humanizer-academic 等)
- 不与 ARIS research-pipeline (CS实验型) 和 academic-pipeline (通用写作型) 冲突，本流水线专注实证计量
- 禁止 mock — S4 数据获取和 S5 计量分析必须真实调用 MCP/Python，产出真实数据和回归表
- 流水线状态文件放项目目录下 (不放 /tmp)，支持断点恢复
- SKILL.md 遵循 Claude Code 标准技能格式 (frontmatter + markdown)

## 设计要点 (建设者参考)

### 7 阶段流水线

```
S1 选题 → S2 文献 → S3 数据 → S4 计量 → S5 写作 → S6 润色 → S7 投稿准备
```

### 每阶段调用的技能

| 阶段 | 核心技能 | MCP/工具 | 交付物 |
|------|---------|----------|--------|
| S1 选题 | research-proposal, academic-paper-strategist | OpenAlex | TOPIC.md (RQ + 假设 + 识别策略) |
| S2 文献 | research-lit (ARIS), literature-review | OpenAlex, Semantic Scholar | LIT_REVIEW.md (文献矩阵 + gap) |
| S3 数据 | csv-data-wrangler, exploratory-data-analysis | FRED, World Bank | DATA_SUMMARY.md + 清洗后数据集 |
| S4 计量 | StatsPAI, pyfixest, marginaleffects | Python 执行 | RESULTS.md (回归表 + 稳健性 + 图) |
| S5 写作 | academic-paper-composer | - | PAPER_DRAFT.md |
| S6 润色 | humanizer-academic, revision-guard | - | PAPER_FINAL.md |
| S7 投稿 | review-paper (自审), latex-document | - | 投稿包 (PDF + cover letter) |
