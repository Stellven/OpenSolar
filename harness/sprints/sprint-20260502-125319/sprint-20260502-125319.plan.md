## 调研发现摘要

### D1: mlx_lm KV cache 抽象

**KVCache** (`/Users/lisihao/FlashMLX/mlx-lm-source/mlx_lm/models/cache.py:522-605`):
- `keys`, `values`: mx.ndarray (GPU unified memory), shape [batch, heads, tokens, head_dim]
- `offset`: 当前 token 数
- `step`: 256 (chunk allocation)
- `update_and_fetch(k, v)`: 核心方法, 追加新 KV 并返回当前完整状态
- `state`: property, 返回 (keys, values) tuple 用于序列化

**RotatingKVCache** (同文件:607-789):
- 增加 `max_size` (sliding window), `_idx` (rotation position), `keep` (保留前 N token)
- `_update_concat()` (batch), `_update_in_place()` (single token)
- 行为: 超过 max_size 时循环覆写, 保留前 `keep` 个 token

**prompt_cache_manager 序列化** (`/Users/lisihao/FlashMLX/mlx-lm-source/mlx_lm/cache.py:132-177`):
1. `save_prompt_cache(file_name, cache, metadata)`:
   - 提取 `c.state` → numpy arrays
   - 提取 `c.meta_state` → metadata (offset, class name)
   - `mx.save_safetensors()` 写磁盘
2. `load_prompt_cache(file_name, model)`:
   - `mx.load_safetensors()` 读回
   - 重建 cache 对象, 恢复 state + meta_state

**KV 数据流**:
```
Token in → Tokenizer → Embedding → Model forward(cache=prompt_cache)
  → 每个 attention layer 调用 cache[layer].update_and_fetch(k, v)
  → KV 写入 GPU unified memory (mx.ndarray)
  → 推理结束: cache 释放, 或 prompt_cache_manager.save() 显式序列化到 safetensors
```

### D4: 内存收益量化

- 35B 4bit 模型: ~20 GB (权重固定)
- 36K context KV: ~735 MB (attention 层, 20 KB/token × 10 层)
- **KV 落盘最多省 735 MB, 模型权重 20 GB 不变**
- KV 占总内存比例: 735 MB / 20735 MB = 3.5%
- **结论: KV 落盘对 35B 模型收益极有限, 模型权重 offload (FlashMLX expert offloading 已实现) 才是大头**
