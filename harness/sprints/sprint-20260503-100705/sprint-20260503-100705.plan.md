## 技术方案

### is_claude_alive_in_pane 递归子树检测

```bash
is_claude_alive_in_pane() {
  local pane_pid="$1"

  # 递归遍历整个子进程树，查找 claude
  local all_descendants
  all_descendants=$(pstree -p "$pane_pid" 2>/dev/null | grep -oE '[0-9]+' | tail -n +1) || {
    # pstree 不可用时 fallback 到 pgrep 递归
    local pids="" queue="$pane_pid"
    while [[ -n "$queue" ]]; do
      local next_queue=""
      for pid in $queue; do
        local children
        children=$(pgrep -P "$pid" 2>/dev/null) || true
        [[ -n "$children" ]] && next_queue="$next_queue $children"
      done
      pids="$pids $next_queue"
      queue="$next_queue"
    done
    all_descendants="$pids"
  }

  for cpid in $all_descendants; do
    [[ -z "$cpid" ]] && continue
    local cmdline
    cmdline=$(cat "/proc/$cpid/cmdline" 2>/dev/null | tr '\0' ' ') || {
      cmdline=$(ps -p "$cpid" -o args= 2>/dev/null) || continue
    }
    case "$cmdline" in
      *claude[\ /]*) return 0 ;;
      claude) return 0 ;;
    esac
  done

  return 1
}
```

注意 macOS 没有 `/proc`，用 `ps -o args=` 获取完整命令行。递归用 BFS 遍历子进程树。

### 熔断限频机制

```bash
# 文件级变量
_last_cb_notify_ts=0

# 在写入 PLANNER-INBOX 熔断通知前
local cb_now
cb_now=$(date +%s)
if (( cb_now - _last_cb_notify_ts >= 300 )); then
  # 写入通知
  printf '...' >> "$HARNESS_DIR/PLANNER-INBOX.md"
  _last_cb_notify_ts=$cb_now
fi
```

### 熔断自动恢复

```bash
# 在 check_panes() 的 rate-limit 读取阶段
# 如果 elapsed >= 600 (AUTO_RECOVER_SECS), 清零 count
local AUTO_RECOVER_SECS=600
if (( elapsed >= AUTO_RECOVER_SECS )); then
  count=0
  # 写回清零状态
  echo "${pidx}:${now}:0" >> "$PANE_RESTART_STATE"

```

## 风险点

1. **macOS pstree 可能不存在**: fallback 到 pgrep 递归 BFS，已考虑
2. **`ps -o args=` 输出格式**: macOS 和 Linux 不同，用 `comm=` 先简单匹配，再用 `args=` 扩展匹配
3. **递归深度**: 正常 claude 进程树深度 3-5 层 (bash→node→claude)，BFS 不会无限递归，因为 pid 是有限的
4. **D3/D5/D6 测试需要真实 tmux + claude 环境**: 如果当前没有运行 harness，测试脚本需要先启动 harness 或模拟环境
5. **不破坏现有功能**: `is_claude_alive` 旧函数保留为 fallback (grep 通过后可移除)，`is_claude_alive_in_pane` 作为新入口
6. **PANE_RESTART_STATE 文件格式**: 是 append-only 的 `pidx:timestamp:count`，去重取最后一行。清零操作也是 append，不影响格式

### 问题 1 (HIGH — 硬伤): D1 伪代码在 macOS 上跑不通

**证据**:

- macOS 没有 `/proc` 文件系统

**问题**: 伪代码 (line 67-101) 用了 `pstree -p` 和 `cat "/proc/$cpid/cmdline"`，这两个在 macOS 都不可用。虽然注释 (line 104) 提到了，但伪代码本身是错的，建设者可能直接复制导致运行时失败。

**修复要求**: 删除 `pstree` 和 `/proc` 分支，**只用** `pgrep -P` BFS 递归 + `ps -p ... -o comm=/args=`。当前 `is_claude_alive` (line 158-179) 的两级 `pgrep -P` 是对的，只需扩展为递归 BFS。
