## Summary

Solar Experience Memory Layer: 借鉴 MIA，把成功/失败 sprint 轨迹压缩成可检索工作流记忆；coordinator 派单前检索经验，避免重复 C-u 风暴、错派 pane、corrupt status、终态 phase 噪音和 builder/evaluator 队列阻塞。 

## When to Use

### Use When
- Coordinator, Planner, Builder, or Evaluator repeats a previously repaired failure class.
- A sprint reaches terminal status but still has misleading phase/handoff metadata.
- Dispatch needs prior success/failure workflow context before selecting role, pane, retry, quarantine, or escalation behavior.

### Do NOT Use When
- The task requires raw full transcript replay rather than compact operational memory.
- The implementation would require online model fine-tuning or external MCP registration.
- The memory store is corrupt or missing; the coordinator must fail open and continue existing dispatch behavior.

## Requirements

- Build a local Experience Memory layer inspired by MIA, storing compact workflows and failure/repair recipes instead of raw conversation logs.
- Extract terminal sprint experience from status.json, events.jsonl, handoff/eval artifacts, quarantine records, and coordinator logs.
- Query experience memory before coordinator dispatch and emit advisory logs without blocking existing behavior.
- Include both success and failure retrieval results; failure memories must prevent repeated prompt residue, C-u storm, stale dispatch, wrong pane, corrupt status, queue/lease, and terminal phase mismatch errors.
- Keep v1 deterministic and inspectable. No online gradient training, no Ruflo init, no MCP registration, no hidden writes to .claude/hooks.

## Scope

### In Scope
- `lib/experience_memory.py` or equivalent local module.
- `solar-harness.sh experience extract|query|doctor` command surface.
- `state/experience-memory/*.jsonl` deterministic store.
- `schemas/experience-memory.schema.json`.
- Coordinator observe-only advisory hook before dispatch.
- Tests and fixtures for known repeated Solar dispatch failures.

### Out of Scope
- Online gradient-based test-time learning.
- Full Ruflo runtime execution, Ruflo init, or MCP registration.
- Deleting or rewriting historical sprint artifacts.
- Replacing coordinator state machine in this sprint.

## Red Flags

- **No mock/stub/TODO**: `grep -rE 'TODO|FIXME|mock|stub|模拟'` in implementation files must return empty
- **No hardcoded secrets**: `grep -rE 'password|secret|token|api.key' --include='*.sh' --include='*.ts' --include='*.py'` must return empty (excluding template placeholders)
- **No temp artifacts**: No important outputs stored in /tmp

## Constraints

- Fail open: memory extraction/query failure must never block dispatch.
- Bounded context: advisory block must stay under 2 KB.
- Deterministic v1 retrieval: use tags/signatures/counts first, no network.
- Security: redact secrets and avoid raw pane transcript storage.
- Observability: every extract/query/advisory event must be logged.
