# Design — Solar Experience Memory Layer

**Sprint**: sprint-20260509-205414
**ADR**: ADR-007
**Status**: Built and Passed (2026-05-10)

## Architecture

Three-layer deterministic local system:

```
┌─────────────────────────────────────────────────┐
│  Coordinator (coordinator.sh)                    │
│  └─ pre_dispatch(sid, action) → Decision         │
│     └─ lib/coordinator_hooks.py                  │
│        └─ experience query (50ms timeout)         │
└──────────────────────┬──────────────────────────┘
                       │ query
                       ▼
┌──────────────────────────────────────────────────┐
│  Query Layer                                      │
│  lib/experience/query.py — query_for_sprint()     │
│  lib/experience/cli.py  — extract|compress|index  │
│                          query|backfill|stats      │
│  └─ SQLite + FTS5 (experience/index.db)            │
└──────────────────────┬──────────────────────────┘
                       │ read entries
                       ▼
┌──────────────────────────────────────────────────┐
│  Processing Pipeline                              │
│                                                    │
│  ┌────────────┐  ┌───────────────┐  ┌───────────┐ │
│  │ Extractor  │→│ Pattern       │→│ Compressor │ │
│  │ (N2)       │  │ Detector (N3)│  │ + Indexer  │ │
│  │            │  │               │  │ (N4)       │ │
│  │ events     │  │ 5 anti-pattern│  │ cluster    │ │
│  │ status     │  │ classes       │  │ by trigger │ │
│  │ handoff    │  │               │  │ + pattern  │ │
│  │ eval       │  │               │  │            │ │
│  └────────────┘  └───────────────┘  └────────────┘ │
│                                                    │
│  Schemas (N1): trajectory + entry JSON schemas     │
└────────────────────────────────────────────────────┘
```

## Data Flow

1. **Extract**: Read terminal sprint artifacts → compact trajectory JSON (secret-stripped)
2. **Detect**: Match trajectory features against 5 anti-pattern classes
3. **Compress**: Cluster trajectories by (trigger_signature, pattern_class) → deduplicated entries
4. **Index**: Upsert entries into SQLite + FTS5
5. **Query**: Coordinator calls pre_dispatch() before each dispatch; returns Decision{allow, deny, advisory}
6. **Audit**: Every decision logged to experience/decisions.jsonl

## Anti-Pattern Classes

| Class | Detection | Default Action |
|-------|-----------|---------------|
| `c_u_storm` | >5 send-keys/C-u events in sprint | advisory |
| `mis_dispatch` | Dispatched but never evaluated | advisory |
| `status_corruption` | status/outcome mismatch, phase/status conflict | advisory |
| `terminal_phase_wake` | Coordinator woke terminal sprint | **abort** |
| `queue_block` | >60min, 0 eval rounds, failure | advisory |

Only `terminal_phase_wake` triggers hard abort at confidence ≥ 0.7. All others are advisory.

## Storage Layout

```
~/.solar/harness/experience/
├── trajectory/<sid>.json      # per-sprint compact records
├── entries/<id>.json          # deduplicated pattern entries
├── index.db                   # SQLite + FTS5
├── decisions.jsonl            # audit log
└── index.db.schema.sql
```

## Safety Properties

- **Fail-open**: Hook timeout 50ms → allow dispatch
- **Bounded context**: Advisory output < 2KB
- **Secret stripping**: Extractor runs regex filter before writing trajectory
- **No raw dumps**: No full pane transcripts stored
- **Kill switch**: `EXPERIENCE_HOOK=0` disables hook entirely
- **Read-only source**: Only reads sprints/, never modifies source artifacts

## Requirement → Node Mapping

| Req | Node | Description |
|-----|------|-------------|
| D1 | N1 | Schema files exist and validate |
| D2 | N2 | Extract CLI writes compact JSONL for terminal sprints |
| D3 | N5 | Query returns success+failure memories |
| D4 | N3 | terminal_phase_mismatch represented as repair_recipe |
| D5 | N7,N8 | Coordinator integration fail-open + bounded |
| D6 | N8 | Safety tests: no raw dumps, secrets, unbounded context |

## Technology Choices

- **SQLite + FTS5**: Already in Solar's stack; no new DB engine; sub-100ms query
- **JSON Schema**: Deterministic validation, no model calls
- **Rule-based patterns**: No LLM summarization in v1; pattern matching on event counts and state transitions
